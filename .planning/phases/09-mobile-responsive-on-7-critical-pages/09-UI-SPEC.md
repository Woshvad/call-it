# Phase 9 — UI Design Contract

> Visual and interaction contract for the 375px-breakpoint mobile retrofit of the 7 critical pages. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is a retrofit, not a greenfield design.** The neobrutalist design system (§14.6) is already shipped and LOCKED. This contract specifies only the *mobile adaptation layer*: how each page stacks at `< 768px`, where full-width buttons apply, touch-target sizing, the hamburger drawer, the desktop-only banner, and the outcome-hero mobile floor. Desktop layouts are unchanged.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (inline-style architecture — no shadcn, no Tailwind utility classes in `apps/web/app`) |
| Preset | not applicable |
| Component library | none (in-house `@call-it/ui` primitives + per-page inline `style={{}}` objects) |
| Icon library | none (inline SVG / unicode glyphs e.g. `×`, `⚔`, `⚠`) |
| Font | Syne (display) · Space Grotesk (body) · JetBrains Mono (mono) — per `packages/ui/tailwind.preset.ts` |

**Responsive mechanism (LOCKED — D-01/D-02):** Mobile adaptation is delivered via a single `useIsMobile()` client hook (`apps/web/app/hooks/useIsMobile.ts`) returning `isMobile`, consumed as `isMobile ? mobileStyle : desktopStyle` inline objects.
- Hook defaults to `true` on the server snapshot AND first client paint (mobile-first first paint, D-02), then converges to the real `matchMedia('(max-width: 767px)')` value after mount (one quiet reflow on desktop).
- NO Tailwind responsive variants (`sm:`/`md:`/`lg:`), NO CSS modules, NO `@media` rules in component styles. `globals.css` `@media` stays reserved for keyframes + `prefers-reduced-motion`.
- Hook lives in `apps/web` ONLY — never in `packages/ui` (would pull `matchMedia` into the Satori/OG build; Pitfall 2).
- Single breakpoint: `< 768px` ⇒ mobile. Tablet (768–1024px) collapses into the desktop bucket (no tablet tier in v1).
- `display: grid` is BANNED (project-wide convention, Pitfall 15). All stacking is `flexDirection: 'row'` → `'column'`.

## Spacing Scale

The shipped frontend uses an ad-hoc inline scale clustered around multiples of 4. Phase 9 preserves the desktop spacing and adds the mobile-only values below. Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, stat-cell label/value gap |
| sm | 8px | Compact inline spacing |
| md | 12px | Mobile button-stack gap, banner/drawer internal gap, mobile gutter half |
| lg | 16px | Mobile page side gutter (`calc(100vw - 32px)` content width), default element spacing |
| xl | 24px | Section padding |
| 2xl | 32px | Section breaks (desktop `marginBottom: 32px` preserved) |
| 3xl | 48px | Major layout breaks |

Exceptions:
- **44px** — minimum touch-target dimension (width AND height) for every interactive element (`button`, `a[href]`, `[role="button"]`, filter chips, tabs, slider thumbs) at `isMobile` ONLY (D-03). Desktop keeps its current dense density (chips/tabs at ~4×10px).
- **16px mobile gutter** — every critical page clamps its container to `width: '100%'` with `padding: '0 16px'` (or `maxWidth: 'calc(100vw - 32px)'`) at `isMobile` so no fixed-width container (`maxWidth: 680/760/1024px`) forces horizontal scroll.

## Typography

Type stack and weights are LOCKED from the design system. Phase 9 introduces NO new fonts or weights. The only typographic change is the **outcome-hero mobile size** (the single most legibility-critical value in this phase, SC3). Roles below describe the mobile-relevant scale:

| Role | Size (desktop → mobile) | Weight | Line Height | Font |
|------|-------------------------|--------|-------------|------|
| Outcome hero | 96px → **52px mobile** | 800 | 1.0 | Syne |
| Heading | 28px (unchanged) | 700 | 1.1 | Space Grotesk |
| Body | 16px (unchanged) | 400 | 1.5 | Space Grotesk |
| Stat / mono value | 15px (unchanged) | 700 | 1.2 | JetBrains Mono |
| Label / meta | 10–11px (unchanged) | 700 | 1.2 | JetBrains Mono / monospace |

**Outcome-hero mobile floor (resolves Open Question 3, Pitfall 4, SC3):**
- Desktop hero is `fontSize: '96px'` Syne weight 800 (`call/[id]/page.tsx:1511`). At 375px this clips the longest variants.
- **Mobile value: `fontSize: '52px'`** applied at `isMobile`. This fits the two longest outcome words — `LOUD AND WRONG` and `FADED CORRECTLY` — on the 343px inner width (375px − 32px gutter) without 2-line wrap, while staying well above the legibility floor.
- The 5 outcome words and their LOCKED colors carry over unchanged: `CALLED IT` (#22C55E green) · `LOUD AND WRONG` (#EF4444 red) · `CONTRARIAN HIT` (#A855F7 purple) · `COLD CALL` (muted) · `FADED CORRECTLY` (#A855F7 purple / accent). Keep `lineHeight: 1.0`, `letterSpacing: '-0.02em'`, the 3px border, hard offset shadow, and `<Stamp>` animation — neobrutalist register preserved (§14.6).
- **Hard floor for the Playwright gate: `fontSize >= 36px` AND the outcome element's bounding box must fit within the viewport (`right <= window.innerWidth`, `left >= 0`, no wrap).** 52px is the design target; 36px is the failing-test floor.
- Add a `data-outcome-word` attribute to the outcome hero `<p>` (`call/[id]/page.tsx` ~line 1509/1518) so the legibility assertion is stable.

## Per-Page Mobile Layout Contract

Each page below specifies the exact `isMobile` adaptation. Desktop is unchanged in every case. Common rule for all 7: container `width: '100%'`, `padding: '0 16px'`, no `minWidth` that exceeds 343px, every `flexDirection: 'row'` two-column block flips to `'column'`, every `flex: 1` action button becomes a full-width stacked button.

### 1. Feed — `apps/web/app/page.tsx`
- Container already `maxWidth: 680px` (narrow). At mobile, clamp to `width: '100%'`, `padding: '0 16px'`.
- **`DuelRowCard` (lines 235/246/353):** the two `minWidth: 120px` cells inside a `flexDirection: 'row'` overflow 343px. At mobile, stack the card to `flexDirection: 'column'` and drop the `minWidth: 120px` (let cells be `width: '100%'`).
- Filter chips already `flexWrap: 'wrap'` — keep, but each chip ≥ 44px tall at mobile (D-03).
- Public-broadcast / call-card rows already single-column — verify no fixed inner width.

### 2. Live Receipt — `apps/web/app/call/[id]/page.tsx` (live branch)
- Container `maxWidth: 1024px` → `width: '100%'`, `padding: '0 16px'` at mobile.
- **Two-column block (activity feed + quote-calls column, ~line 1656):** `flexDirection: 'row'` → `'column'` at mobile (activity feed first, quote-calls below).
- **Action button row (Follow / Fade / Challenge):** stack full-width (`flexDirection: 'column'`, each button `width: '100%'`, `gap: 12px`), ≥ 44px tall. Preserve the fill/outline/orange-outline treatments.
- The quote-calls column here is a read-only DISPLAY surface (NOT the composer) — it stays in scope as part of this responsive page; it does NOT get a desktop-only banner.

### 3. Settled Receipt — `apps/web/app/call/[id]/page.tsx` (settled branch) — **#1 SHARE-LANDING PRIORITY**
- Container `maxWidth: 1024px` → `width: '100%'`, `padding: '0 16px'` at mobile.
- **Outcome hero (line 1511):** `96px` → `52px` at mobile (see Typography). Add `data-outcome-word`.
- **4-stat row (line 1567 — FINAL VALUE / TARGET / CONVICTION / P&L, 4 × `flex: 1`):** **stack as 2×2 at mobile** (resolves Open Question 2). Use `flexWrap: 'wrap'` with each cell `flex: '1 1 45%'` (or `width: '50%'`) so two cells per row. Preserve the 1px inter-cell dividers and the 2px outer border (neobrutalist). Rationale: 4 cells × ~85px is borderline-cramped at 343px; 2×2 keeps each value readable and preserves the bordered-grid look without `display: grid`.
- **Action row (line 1596 — Share / Share as Frame / View All Calls, `flex: 1`):** stack full-width (`flexDirection: 'column'`, each `width: '100%'`, `gap: 12px`), ≥ 44px tall (UI-20 "full-width stacked on mobile"). Preserve the `boxShadow: '4px 4px 0 0 #09090E'` hard offset on the primary.
- FINAL POSITIONS block + provenance line + Caller-Exited / Disputed variants: any `flexDirection: 'row'` stat block stacks to `column`.

### 4. Profile — `apps/web/app/profile/[address]/ProfileClient.tsx` (real surface)
- Container `maxWidth: 680px` → `width: '100%'`, `padding: '0 16px'`.
- Stat row (line 114) already `flexWrap: 'wrap'` — keep, ensure ≥ 44px tap targets where interactive.
- **CATEGORY REPUTATION grid (lines 128–130 — 3 cards `flex: '1 1 0' minWidth: 160px`):** 3 × 160 = 480 > 343 → force `flexDirection: 'column'` at mobile (each card `width: '100%'`, drop `minWidth`).
- `apps/web/components/ProfileTabs.tsx` tab row: tabs ≥ 44px tall at mobile; allow `flexWrap: 'wrap'` if they exceed width.

### 5. Leaderboard — `apps/web/app/leaderboard/LeaderboardClient.tsx` (real surface)
- Container `maxWidth: 760px` → `width: '100%'`, `padding: '0 16px'`.
- **Mobile column set (resolves Open Question 1, UI-13):** the desktop fixed-width table (`#` / `CALLER` / `REP` / `ACC` / `BEST` category / `Δ7D` sparkline; `width: 48/80/72px` cells, lines 218–276) overflows 343px. At mobile, render a **3-column compact row: `#` · `CALLER` (handle + VERIFIED badge, flexes to fill) · `REP`**. DROP `ACC`, `BEST category`, and the `Δ7D sparkline` columns at mobile (they remain on desktop). No horizontal scroll. **The viewer's-own-row accent highlight MUST survive** the mobile collapse. Do not introduce `display: grid` — keep the row as `flexDirection: 'row'` with the dropped cells removed and `CALLER` taking remaining width.
- Each row ≥ 44px tall (it links to a profile — D-03).

### 6. Sign-in — `apps/web/app/signin/page.tsx` (+ `SignInButtons.tsx`)
- Already centered (`flexDirection: 'column'`, `maxWidth: 400px`, `minHeight: 100vh`, buttons `width: 100%`) — lightest page.
- At mobile, clamp the 480px frame / 4px corner brackets (UI-24) so they don't overflow: outer frame `maxWidth: 'calc(100vw - 32px)'`.
- The 3 sign-in CTAs already full-width — ensure ≥ 44px tall.

### 7. Onboarding — `apps/web/app/onboarding/layout.tsx` + 5 subroutes
- Layout `maxWidth: 480px` → add `width: '100%'`, `padding: '0 16px'` so the 480px frame clamps at 375px.
- Verify EACH of the 5 subroutes individually (`handle`, `socials`, `follow-graph`, `fund`, `tagline`): forms already `flexDirection: 'column'` with `width: 100%` inputs — confirm inputs and advance buttons are ≥ 44px tall and the tagline commitment line wraps cleanly.

### In-scope modals / shared components (D-04) — at 375px
Universal modal rule: panel `maxWidth: 'calc(100vw - 32px)'`, overlay has horizontal padding so the panel never touches screen edges, internal `flexDirection: 'row'` button rows stack to `column` full-width, all buttons ≥ 44px tall.

| Component | File | Mobile action |
|-----------|------|---------------|
| FollowFadeModal | `packages/ui/src/compound/FollowFadeModal.tsx` | Already `maxWidth: calc(100vw - 32px)` — verify internal button rows stack. Apply mobile styling via props/page wrapper, NOT inside the shared component (OG/Satori coupling, Pitfall 2). |
| CallerExitModal | `packages/ui/src/compound/CallerExitModal.tsx` | Add the `calc(100vw - 32px)` clamp to match FollowFadeModal. |
| PositionExitModal | `packages/ui/src/compound/PositionExitModal.tsx` | Add the `calc(100vw - 32px)` clamp. |
| ChallengeFormModal | `apps/web/app/components/ChallengeFormModal.tsx` | Panel `maxWidth: 480px` → ensure overlay h-padding; stack internal `flexDirection: 'row'` rows. |
| DisputeModal | inline `call/[id]/page.tsx` (~line 463/600) | `maxWidth: 480px` → add `calc(100vw - 32px)` clamp. |
| ProvenanceModal | inline `call/[id]/page.tsx` (~line 784/818) | `maxWidth: 560px` **overflows 375px** → MUST add `maxWidth: 'calc(100vw - 32px)'`. |
