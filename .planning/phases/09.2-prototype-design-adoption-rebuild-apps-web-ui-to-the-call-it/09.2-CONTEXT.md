# Phase 09.2: Prototype Design Adoption - Context

**Gathered:** 2026-06-10
**Status:** Ready for planning
**Source:** Operator decisions captured live in-session (2026-06-10) after the 8-agent compatibility audit (09.2-RESEARCH.md). The operator stated: "I want everything to be linked to this prototype design, that was what I wanted from the beginning." The prototype at `call it frontend/` is the design of record for the whole product.

<domain>
## Phase Boundary

Rebuild every user-facing surface of `apps/web` to render the canonical design prototype at `call it frontend/` (ROOT variant). This is a **restyle-only phase**: markup, tokens, fonts, and CSS change; hooks, data sources, guards, contracts, relayer, and subgraph do NOT. No new backend endpoints are built in this phase — design elements whose data doesn't exist yet degrade to hidden. Phase 09.1 (testnet demo hardening) runs AFTER this phase, polishing the NEW design.

</domain>

<decisions>
## Implementation Decisions

### Design canon
- D-01: The ROOT variant of `call it frontend/` is the design canon for ALL surfaces (Archivo 800/900 display + Inter body + JetBrains Mono data; cream `#F5F1E8` inverse blocks; hard black offset shadows 4/8/2px zero-blur; border-radius 0 everywhere; 2-3px borders; uppercase tracked display type). `v1-clean/` is a UX reference ONLY for create-flow details — never its rounded/Space Grotesk skin.
- D-02: Prototype `:root` tokens (`call it frontend/styles.css:3-41`) become the single token layer in `apps/web/app/globals.css` as CSS custom properties. `packages/ui/tailwind.preset.ts` colors remap to `var(--*)` KEEPING the existing `brand-*` alias names so ~30 `@call-it/ui` consumer files keep compiling. The three legacy gray palettes (`#18181B/#27272A/#A1A1AA/#2A2A30`) are retired by end of phase.
- D-03: Money/outcome colors: prototype chartreuse win `#E8F542` and soft red loss `#F87171` REPLACE the app's green/red (`#22C55E`/`#EF4444`). CONTRARIAN HIT stamp uses the win color (this also fixes the documented stale `outcome-contrarian #A855F7` token noted in `Stamp.tsx:13-15` vs spec §14.1). `#A855F7` remains the duel/challenger identity color.
- D-04: Fonts Archivo + Inter + JetBrains Mono load via `next/font` (brand fonts never rendered in-browser before, so this is net-new, not a swap). OG images KEEP Syne/SpaceGrotesk TTFs in `apps/web/app/fonts/` for now — UI/OG font divergence is accepted in-phase; OG Archivo refresh is the optional final wave.
- D-05: The prototype is a **markup + token donor ONLY, never a logic donor**. Every screen keeps its existing hooks, data fetching, tx writes, validation, and auth gating. Do not port prototype state, mock data, hash routing, or `tweaks-panel.jsx`.

### Interaction model
- D-06: Follow/fade stays amount-based via the real `FollowFadeModal` — the prototype's one-click follow/fade toggles are NOT adopted (money flows are real). Design CTAs open the existing modals.
- D-07: Design data with no live source degrades to HIDDEN, never faked: feed pool $ and follow/fade/challenge counts, caller rep/accuracy/streak on feed cards, leaderboard sparklines/7d-deltas/best-category/badges/hero StatBlocks, profile rep-history/calibration/ROI/category bars/followers/notable receipts, platform totals, duel live spread, settled-timeline middle milestones (ship 2-node created→settled), hardcoded block/tx footers (use real provenance data where it exists).
- D-08: Dead prototype controls (SORT, non-filtering category chips, 7D/30D period toggles that don't refetch, NEXT-10 pagination, leaderboard row click, collapse toggles) are CUT (not rendered) unless they wire trivially to existing data. No dead buttons ship.
- D-09: The settled-receipt SHARE control keeps the app's existing working share wiring (warpcast/twitter intents + SHARE AS FRAME) — the prototype's dead SHARE button is superseded.

### Shell & routing
- D-10: New app shell: 64px sticky header (brand + tagline, ⌘K search placeholder allowed as visual, wallet USDC pill from existing `useUsdcBalance`, existing NotificationBell) + 240px sidebar (Next.js links, //TAPE //YOU sections, cream YOUR REP card fed by existing viewer profile data — no sparkline v1) replacing the 52px GlobalNav on desktop. MobileDrawer remains the <768px nav (sidebar collapses into it). The shell is built AROUND `ClientProviders` — never inside `Providers.tsx`; the locked provider order and its AST test stay untouched.
- D-11: SystemTicker (32px price tape) ships behind a feature flag, default OFF (no price endpoint exists). Layout must not hardcode the ticker's 32px offset when the flag is off.
- D-12: Landing: adopt the prototype `home.jsx` design as the logged-out experience. Implementation choice (public `/` with middleware carve-out rendering landing for unauthenticated users, vs restyled `/signin` carrying the hero) is Claude's discretion — but middleware public-prefix behavior, the privy-token cookie flow, and the onboarding redirect chain MUST keep working, and the existing signin Playwright hooks (`data-testid` signed-in, disclaimer) must survive.

### Coverage & invariants
- D-13: ALL surfaces get the new design — including those the prototype doesn't cover: signin, onboarding (5 steps + progress dots), disputes page + DisputeModal/ProvenanceModal, settings, terms, notifications (bell + inbox), quote composer (`/new?quote=`), ChallengeFormModal, FollowFadeModal/CallerExitModal/PositionExitModal, PaymasterCapBanner/WalletExportPrompt, DesktopOnlyBanner. The prototype's unused cream `.modal-overlay/.modal-panel` (`styles.css:821-840`) is the template for all modals; `.brutal-input/.toggle-pill` for forms; `.activity-row` for the inbox.
- D-14: Hard invariants in every new markup: AUTH-44 (handles only, never wallet addresses), D-09 fader guard via `lib/outcome-word.ts` (words come from the app, colors from prototype `SETTLED_OUTCOMES` at `receipt.jsx:284-290` — NEVER from `data.jsx` OUTCOMES, which references broken CSS vars), contract addresses only from `@call-it/shared`, `MiniAppReady` mounted on every `/call/[id]` branch, flexbox-only in `packages/ui` and all OG paths (Satori grid ban, lint-enforced), Tailwind stays v3.4 classic with `postcss.config.js` (NO v4 `@theme` port), `@farcaster/auth-kit/styles.css` stays in the cascade, existing Playwright `data-testid` hooks preserved.
- D-15: Tests are part of the phase: visual goldens (visual-smoke, design-system-snap, og-thumbnail-200px, responsive) get regenerated against the new design as an explicit final-wave task; source-assert tests (`leaderboard.spec.ts:63` `#1A1A24` literal, `new-call-publish.spec.ts:103` ConvictionBar import, profile `@call-it/ui` import asserts) are updated deliberately, never deleted to pass. Full web vitest + build must exit 0 at every wave boundary.
- D-16: Adoption sequence follows 09.2-RESEARCH.md §6: (0) tokens + `next/font` + primitive classes + `@call-it/ui` retheme + shell → (1) leaderboard → profile → feed → (2) `/call/[id]` live + settled receipt branches wholesale (keep data/handler block lines ~95-1390) + all 5 modals → (3) `/new` restyle over existing RHF/zod/hooks + conviction-zone words (Hesitant/Confident/Bold/On record) + $5/25/50/100 quick-picks → (4) duel + landing + uncovered surfaces → (5) golden regeneration + mobile clamp pass (+ optional OG Archivo refresh).

### Claude's Discretion
- Tailwind utility classes vs ported plain-CSS classes per component (both acceptable; primitives in globals.css are plain CSS).
- Component file organization for the new shell (e.g., `AppShell.tsx`, `Sidebar.tsx`, `SystemTicker.tsx` placement).
- Exact mobile clamps for the 56-132px display type and 4-5-col stat rows (prototype has zero @media — mobile interpretation is yours, preserving Phase 9's 375px requirements).
- Whether the OG Archivo refresh lands in-phase (optional final wave) or defers to 09.1.
- v1-clean create-flow nice-to-haves (drafts/autosave, shorthand parsing, similar-calls modal) — backlog by default, include only if cheap.

</decisions>

<specifics>
## Specific Ideas

- Voice/copy follows the prototype: "Go on record.", "Be right in public.", "MARK IT.", "permanent on settle", interpunct "·" as the universal separator, tiny letter-spaced UPPERCASE MONOSPACE for every label/microcopy.
- Cream CTA press physics: translate 2px/4px on hover/active with shrinking hard shadow, 0.12s linear transitions.
- OutcomeStamp: 120px Archivo 900, rotate(-1deg), text-shadow 4px 4px 0 #000, stampReveal overshoot animation (the app's framer-motion Stamp maps to this).
- Avatar grad palette a-f: #E8F542/#FB923C/#A855F7/#22D3EE/#F472B6/#FBBF24; avatars become square (radius 0).
- Sidebar footer microcopy style: "V0.4.1 · ARBITRUM ..." mono footer (use real values; Sepolia in current deploy).
- Wallet pill format: "2,840.21 USDC" with avatar square.

</specifics>
