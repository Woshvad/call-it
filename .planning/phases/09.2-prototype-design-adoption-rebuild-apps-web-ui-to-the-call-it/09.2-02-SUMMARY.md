## Accomplishments

- **Button** rewritten to the prototype `.btn` recipe: Archivo (font-display) 700 13px uppercase 0.04em, 2px border, radius 0, 0.08s linear transitions. 8 intent keys: `primary`/`cream` = the signature cream CTA (bg `brand-cream`/var(--bg-inverse), black text, 2px black border, `shadow-[4px_4px_0_0_#000]`, weight 800) with DOWN-RIGHT press physics (hover `translate(2px,2px)` + 2px shadow; active `translate(4px,4px)` + `shadow-none` — the opposite of the legacy up-left hover); `secondary`/`outline-white` = transparent + 2px white border; `danger`/`fade` = loss outline (var(--accent-loss)); `duel` = #A855F7 outline (duel identity only, D-03); `ghost` = borderless text-secondary. New `big` size (18px/28px padding, 15px, 3px border). Accent chartreuse appears on NO button background (UI-SPEC reservation)
- **Card** carries `.brutal-card`: bg `var(--bg-secondary)`, 2px var(--border-subtle), p-6, radius 0, plus variants `heavy`/`hero` (3px white border + brutal black shadow, hero p-8), `cream` (inverse panel), `interactive` (hover white border + shadow + translate(-2px,-2px)). The legacy `shadow-[6px_6px_0_0_#E8F542]` accent shadow is DELETED; the `accent` prop stays functional as a 3px `var(--border-accent)` border (leaderboard.spec `<Card accent` consumers compile untouched)
- **Tag/VerifiedBadge** carry the `.pill` recipe: JBM (font-mono) 11px 600 uppercase 0.06em, 3px/8px padding, 1px intent-colored border, inline-flex only (Satori parity); pill variant intents `win/loss/duel/neutral/muted` added alongside the original four keys; VerifiedBadge API and `border-brand-accent` rendering unchanged
- **CornerBrackets** = `.bracketed` recipe: 14px L-shaped 1px `var(--border-active)` marks at 8px inset (was 16px 4px accent)
- **Stamp** rethemed to `.outcome-stamp`: `clamp(56px, 17vw, 120px)` Archivo 900 uppercase, line-height 0.85, `rotate(-1deg)` (was -5deg), `text-shadow 4px 4px 0 #000`, framer-motion stampReveal overshoot retimed to 0.4s `cubic-bezier(0.34,1.56,0.64,1)` with scale keyframes 1.3→0.96→1 and rotate -2deg→-1deg. KEPT: `useReducedMotion` fallback, `hexColor` override, COLOR_MAP lookup — which since 09.2-01 returns #E8F542 for `outcome-contrarian`, so the CONTRARIAN HIT stamp path resolves to the win color through the token map itself (D-03 fixed; stale-token comment replaced with the fix record)
- **ConvictionBar**: Radix slider logic + muted→accent fill interpolation preserved verbatim (UI-51); chrome → `.brutal-bar` 10px track (1px border, radius 0, var(--bg-tertiary)) + `.brutal-slider` 26px cream square thumb (2px black border, `2px 2px 0 #000`)
- **MarketPositioningBar**: `.brutal-bar.split` — follow side `var(--accent-win)`, 2px black gap segment (`flex: 0 0 2px`), fade side `var(--accent-loss)`, fills transition `width 0.4s cubic-bezier(0.16,1,0.3,1)`; all props, formatUsdc, and percentage logic unchanged; flexbox-only

## Decisions Made

- Button `danger` = loss outline (donor `.btn.fade` recipe) rather than the legacy solid `bg-outcome-loss` — the prototype has no solid red button; cva test updated in lockstep
- Aliased intents (`cream`≡`primary`, `outline-white`≡`secondary`, `fade`≡`danger`) share recipe constants so they stay byte-identical (asserted by a new test)
- Stamp's boxShadow-expansion animation (UI-45 legacy) replaced by the prototype's static `text-shadow 4px 4px 0 #000` — `.outcome-stamp` is a text stamp, not a box
- Skeleton pulse implemented with the existing `liveDot` keyframe at a slow 2s cadence — avoids new keyframes (globals.css not in this plan's files) and avoids the banned `animate-pulse` utility string
- MarketPositioningBar inline styles use `var(--*)` tokens (acceptance-sanctioned); OG/Satori routes build their own markup and are unaffected
