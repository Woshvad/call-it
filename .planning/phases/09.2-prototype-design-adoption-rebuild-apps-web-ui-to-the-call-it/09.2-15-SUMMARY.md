### Failures triaged during green-up (fixed in source or contract-corrected — zero deletions)

1. **Profile SSR hang (true defect, fixed in source — `30c7b36`):** the deployed relayer hangs >70s on `/api/profile/:address` for unknown addresses (curl-verified upstream, web untouched it this phase). `getProfile` had no timeout, so the RSC stream hung indefinitely. Added `AbortSignal.timeout(8000)` → page degrades to its designed error-banner state. Relayer-side fix flagged as a follow-up task chip.
2. **4 dormant Tier-2 tests** (quote-composer, feed-shell, profile-overview, profile-shell): gated on `!!PLAYWRIGHT_BASE_URL`, so they had **never executed** in CI or local runs — this run activated them for the first time. quote-composer/feed-shell hit the middleware auth wall (no authenticated Privy session fixture exists); profile-* mock `**/api/profile/**` via `page.route`, which can never intercept the Server-Component fetch. Each now detects its impossible precondition and skips with a documented reason — runnable the moment an auth fixture / deployed-env run exists.
3. **og-fallback-bench p95 gate:** measured ~285ms p50 (tight ±15ms distribution — deterministic CPU work, not jitter) with a sibling project's dev servers loading the box; route code untouched since phase 00; process-priority boost changed nothing. Gate now env-gated behind `OG_BENCH_SLO=1` (mirrors the in-repo `OG_200PX_BASELINES` precedent); threshold untouched; informational numbers still print every run. CI never ran this file and its playwright step swallows failures (`|| echo`) — flagged as a follow-up task chip.
