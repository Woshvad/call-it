## 1. WHAT THE PROTOTYPE IS

A **zero-build, browser-transpiled React prototype**: `Call It.html` loads React 18 UMD + Babel-standalone; every `.jsx` runs as `<script type="text/babel">` sharing one `window.*` namespace; hand-rolled hash routing (`app.jsx:111-165`); all data is mock constants; **all interactivity is cosmetic** (no submit handlers, no real follow/fade transactions). Design system = one plain-CSS file (`styles.css`, 918 lines, CSS custom properties, **zero `@media` queries — desktop-only**). `tweaks-panel.jsx` is prototype-host scaffolding — do NOT port; its knobs are the canonical enumeration of designed variants (accent swap, ticker on/off, 5 outcome states, callerExited, settledCalls gate).

**Variant canon (two sequential design eras, not branches):**
- **ROOT is design canon for everything**: `styles.css` header says "Neo-Brutalist + Alpha redesign"; it restyled every v1-clean screen (Archivo/Inter, cream `#F5F1E8` inverse blocks, hard 4px black shadows, radius 0), added the landing page (`screens/home.jsx`, 451 lines), the callerExited variant, SETTLED_OUTCOMES, winners/losers, duel riders, leaderboard chips.
- **v1-clean is functional reference ONLY for the create flow** (`v1-clean/screens/create.jsx` 1204 + `create-shared.jsx` 775 + `create-event.jsx` 610: per-type subforms, duplicate-hash gate, ConfirmModal, autosave/drafts, token-based live preview, shorthand parsing). Root's `screens/create.jsx` is a 470-line visual shell ("(brutalist)" comment, price/spread have no fields). **Critically: `apps/web/app/new/` already implements most of v1-clean's logic in production form** (RHF+zod, dup-check, preflight, conviction gate, PublishConfirmModal, live Receipt preview) — so the create adoption is "root visuals over existing app logic", with v1-clean as UX reference for details (drafts/autosave, conviction-zone words, shorthand parsing), not a code port.
- **Screenshots span both eras** (`screenshots/`): `home.png` + `settled.png` = ROOT ground truth; all create/modal frames = v1-clean skin (rounded corners, Space Grotesk) — **do not copy their soft radii/fonts**; copy only their flows/states (drafts counter, disabled publish CTA, modal scrim+blur, conviction-zone highlight).

## 5. WIRING-RISK REGISTER

1. **Cascade is dead on arrival for a drop-in stylesheet.** ~700+ inline `style={{}}` objects (call page 224, duel 104, home 69) win over any global CSS; `app/layout.tsx:24-32` sets body bg/color/font inline. Strategy: token layer + primitives first, then **per-file markup rewrites** route-by-route. Never fight inline styles with `!important`.
2. **Fonts.** Adopt Archivo/Inter via `next/font` (brand fonts never render today, so in-browser swap is cheap). BUT OG images use Syne/SpaceGrotesk TTFs from `app/fonts/` (must STAY there — `tests/og-unit.test.ts:125-132`); UI and OG will diverge until an OG refresh adds Archivo TTFs to `lib/og-fonts.ts`. Accept divergence short-term.
3. **Nav-model mismatch.** App: 52px GlobalNav with NO desktop links + MobileDrawer. Prototype: 32px ticker + 64px header + 240px sidebar with hardcoded sticky offsets (sidebar `top:96px` assumes the ticker). Sidebar shell is net-new; ticker needs a price endpoint (relayer/Hermes — none exists; ship behind a flag or defer). Keep middleware public-prefix behavior if any route moves.
4. **Client/server constraints.** Hash routing → Next `Link`/router; `Countdown` setInterval, keyboard shortcuts, ticker = client components; `/leaderboard` + `/profile` must stay Server-Component-page → props → client-renderer; Privy mounts via `ssr:false` so the new shell must tolerate a late-hydration flash; provider order is AST-locked (`tests/privy-provider-order.ast.test.ts`) — build the shell AROUND `ClientProviders`, never inside it.
5. **Satori/grid ban.** Prototype layouts use CSS grid heavily; fine in pages, **banned in `@call-it/ui` and OG paths** (`eslint-rules/no-display-grid-in-og.js`, error-level; Receipt↔`og-fallback-render.ts` parity contract). Any prototype component promoted into `packages/ui` must be re-laid-out in flexbox.
6. **Tests that WILL trip** — plan regeneration, don't be surprised: visual goldens (`visual-smoke`, `design-system-snap`, `og-thumbnail-200px`, `responsive`); source-asserts: `tests/leaderboard.spec.ts:63` requires literal `'#1A1A24'` in LeaderboardClient source (happily, that's a prototype token), `new-call-publish.spec.ts:103` ConvictionBar import, profile specs assert `@call-it/ui` imports.

Port outcome colors from `receipt.jsx:284-290` SETTLED_OUTCOMES; words come from the app's `lib/outcome-word.ts` (already correct, incl. D-09 per-viewer FADED CORRECTLY guard).
8. **Mock fields with no data source** (do not fake): feed pool $ (`follows*12/fades*14` is fabricated — real pools exist only via FFM reserves on the call page), feed follow/fade/challenge counts, caller rep/streak/accuracy in feed cards, leaderboard sparklines/7d deltas/best-category/badges, profile rep-history/calibration/ROI/category-reps/followers/notable-receipts, hero rank-01 metrics, platform totals, ticker prices, duel live spread, settled-timeline milestones, hardcoded blocks/tx-hashes (real ones: provenance endpoint + subgraph). 10. **Invariants to preserve in all new markup:** AUTH-44 (handles only, never wallet addresses), D-09 fader guard, addresses only from `@call-it/shared`, MiniAppReady on every `/call/[id]` branch, Playwright `data-testid` hooks (signed-in, disclaimer, onboarding dots), flexbox-only in shared/OG components, Tailwind stays v3.4 (`postcss.config.js` mandatory — do NOT attempt a v4-style `@theme` port), `@farcaster/auth-kit/styles.css` remains in cascade.

## Validation Architecture

**Existing test infrastructure (all in place — no Wave 0 framework install needed):**

| Surface | Framework | Command | Notes |
|---|---|---|---|
| Unit/invariant tests (web) | vitest | `pnpm --filter @call-it/web test` | ~97 tests incl. provider-order AST lock, AUTH-44 invariant, og-unit font asserts |
| UI package | vitest | `pnpm --filter @call-it/ui test` | primitive component tests |
| Build gate | next build | `pnpm --filter @call-it/shared build && pnpm --filter @call-it/web build` | shared MUST build first (dist gitignored) |
| Lint (incl. Satori grid ban) | eslint | `pnpm --filter @call-it/web lint` | `no-display-grid-in-og` is error-level |
| E2E + visual goldens | Playwright | `pnpm --filter @call-it/web exec playwright test` | visual-smoke, design-system-snap (needs `NEXT_PUBLIC_DEV_ROUTES=1`), responsive, leaderboard, new-call-publish, receipt-meta, og-thumbnail-200px (env-gated `OG_200PX_BASELINES=1`) |

**Validation strategy for a restyle phase:**
1. **Per-task quick gate:** `pnpm --filter @call-it/web test` (vitest, no server needed) + targeted `tsc`/build when a task touches the token/preset layer. Invariant tests (AST provider order, AUTH-44, outcome-word D-08/D-09, og-unit) are the tripwires that prove "markup donor only" was honored.
2. **Per-wave gate:** full `@call-it/shared` → `@call-it/ui` → `@call-it/web` builds exit 0 + full web vitest + lint.
3. **Visual goldens are EXPECTED to break** — they encode the OLD design. They are regenerated deliberately in the final wave (`--update-snapshots`) after the operator visually approves the new design; never regenerated mid-phase to silence a failure.
4. **Source-assert tests** (leaderboard `#1A1A24` literal — happily a prototype token; ConvictionBar import; `@call-it/ui` import asserts) are UPDATED in the same task that changes the asserted source, keeping each task's test run green.
5. **Manual-only verifications:** side-by-side fidelity check vs `screenshots/home.png` + `settled.png` ground truth; cream press physics + stampReveal animation feel; late-hydration flash with the new shell (Privy `ssr:false`); mobile drawer fold at 767px.

**Max feedback latency:** ~90s (web vitest) per task; ~5min (builds + vitest + lint) per wave.
