- Deleted (not conditionally hid) the fake category bars — D-07 distinguishes "no source yet" (hide) from "fabricated" (remove); these were fabricated
- The null-profile truncated-handle fallback was an AUTH-44 violation hiding in the old code — removed as part of the port rather than carried over
- ProfileHeader keeps consuming the `stats` prop but renders it as the prototype's JBM interpunct metadata line instead of a separate stats row — chrome change only, zero API change

- `handle` with relayer source `'truncated'` still renders (pre-existing AUTH-11 final-resolution behavior upstream of this plan's files) — consistent with the 09.2-04 leaderboard acceptance of the same lib behavior
