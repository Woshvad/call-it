## Decisions Made

- **Trending-duel pin cut, not kept**: the plan allowed the pin to stay "ONLY because it already degrades to hidden". Verified instead that `/api/duels` has no route at all (`apps/web/app/api` = frame/ + og/ only) — the pin was not "degrading", it was permanently unreachable, and keeping it meant retaining ~650 lines of dead legacy-styled code (DuelRowCard, three badges, two fetchers) that would itself have needed retheming. D-08's no-dead-controls rule applied to the whole path
- **Challenge affordances removed from the feed**: `openToChallenges` is absent from the relayer `/api/feed` response (grep-verified), so the Challengeable chip always filtered to empty and the per-card CTA was permanently disabled. The real challenge flow on `/call/[id]` is one tap away via the new card navigation
- **Live tab filters out settled items** (previously showed everything) — matches the tab label with one honest line of filtering
- **Empty/error states owned by page.tsx** so the acceptance strings live there; FeedList returns null on empty (header contract documented)
- **`.card-enter` kept over `.stagger`** — already consistently applied with per-item `--index`, keeps feed-shell Tests 3/5 and the reduced-motion fallback intact
