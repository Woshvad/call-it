### 5. `@call-it/ui` Button — the CVA pattern (THE primitive template)

**Analog:** `packages/ui/src/primitives/Button.tsx` (full file, 57 lines). The pattern every new/rethemed primitive must follow — `cva(baseArray, { variants: { intent, size }, defaultVariants })` + `cn()` + `VariantProps` typing:

```tsx
const button = cva(
  ['inline-flex items-center justify-center', 'font-body font-semibold',
   'border-2 border-black', 'shadow-[4px_4px_0_0_#000]',
   'transition-transform duration-100 ease-out', /* hover/active translate */],
  { variants: { intent: { primary: 'bg-brand-accent text-black border-black', ... },
                size: { sm: 'px-3 py-1.5 text-xs', md: '...', lg: '...' } },
    defaultVariants: { intent: 'primary', size: 'md' } });
export type ButtonProps = React.ButtonHTMLAttributes<HTMLButtonElement> & VariantProps<typeof button>;
export function Button({ intent, size, className, children, ...props }: ButtonProps) {
  return <button className={cn(button({ intent, size }), className)} {...props}>{children}</button>; }
```

Retheme: base → `font-display font-bold uppercase tracking-[0.04em]` (prototype .btn:387-403); add intents `cream | fade | duel | outline-white | ghost` mapping the `.btn.*` recipes above (fade = transparent + `--accent-loss` border; duel = `#A855F7`; cream = press-physics signature). Keep existing `primary/secondary/danger` names working or alias them — consumers reference them.

Card.tsx analog (lines 15-34): swap base to `.brutal-card` recipe — `border-2 border-[var(--border-subtle)] bg-[var(--bg-secondary)] p-6`; variants `heavy | hero | cream | interactive`; the current `shadow-[6px_6px_0_0_#E8F542]` accent shadow DIES (prototype shadows are all-black, §2 delta). Keep the `accent` prop functional (leaderboard.spec Test 4 asserts `<Card accent`).

Stamp.tsx analog: the framer-motion overshoot + `useReducedMotion` fallback + `hexColor` override pattern is already correct — retheme to prototype OutcomeStamp: 120px clamp Archivo 900, `rotate(-1deg)` (currently -5deg), `text-shadow: 4px 4px 0 #000`. The `hexColor` prop (lines 31-55) is the mechanism that already bypasses the stale `#A855F7` contrarian token — D-03's color fix flows naturally once `COLOR_MAP`/preset are updated.
