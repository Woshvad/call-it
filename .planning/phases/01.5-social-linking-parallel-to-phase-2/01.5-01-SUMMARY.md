# Phase 01.5 Plan 01: Relayer Authorization + Social-Linking Env Surface Summary

**The single hard gate for the CORE wave: extended the relayer/shared env schema with the social-linking surface, then drove the owner-gated `setRelayer(relayerKmsAddress)` on the canonical ProfileRegistry so `linkTwitter`/`linkFarcaster` stop reverting `NotRelayer`. On-chain `relayer()` now returns the oauth-proof KMS wallet `0xdFc80922…`.**

## Accomplishments

- **Task 1 — env surface (`3fbf88d`):** appended the Phase 1.5 social-linking fields to `EnvConfigSchema` (all `.optional()`, network-agnostic): `RELAYER_OAUTH_PROOF_ADDRESS` (0x-address regex), `FARCASTER_RELAY_URL` (url), `FARCASTER_AUTH_DOMAIN` (min 1), plus the FEED-wave placeholders `X_API_BEARER_TOKEN` / `NEYNAR_API_KEY`. Mirrored into `RelayerEnv` (`types.ts`) and the `loadSecrets()` return (`secret-manager.ts`). `@call-it/shared` + `@call-it/relayer` build exit 0.

- **Task 2 — `setRelayer` checkpoint CLEARED (operator session, 2026-06-07):**
  1. **Derived the target address** with the no-key helper `apps/relayer/src/scripts/prepare-set-relayer.ts` (commit `9b41f0f`) — reads the `oauth-proof` GCP KMS **public** key (no key material, no signing) and computes its Ethereum address: **`0xdFc80922FAbc51a08350c0b371917e6EaB8b550A`**. The same script confirmed the pre-state (`relayer()` == `0x0`, `owner()` == treasury `0xDa8c5726`).
  2. **GCP ADC** was set up (`gcloud auth application-default login`, project `call-it-sepolia-602217`) so the derivation could read the KMS public key.
  3. **Funded** the KMS wallet with 0.05 Sepolia ETH from the treasury (relayer pays link gas, D-03).
  4. **Broadcast** `setRelayer(0xdFc80922…)` on the canonical PR `0xF66C0AFEf03b43338FC5aE282e45C0Cf6A3c4820` from the treasury owner key `0xDa8c5726` (keystore-imported, address-verified before signing).
  5. **Set `RELAYER_OAUTH_PROOF_ADDRESS=0xdFc80922…`** in the local `apps/relayer/.env.local` (gitignored) to match the on-chain value (Pitfall 1).
