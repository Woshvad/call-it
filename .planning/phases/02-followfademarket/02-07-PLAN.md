<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated user→GET /api/notifications | Must only return notifications for the authenticated user's address |
| Authenticated user→POST /mark-read | Must only mark notifications belonging to the authenticated user |
| Public→GET /api/calls/:id/live-state | Unauthenticated; read-only; no private data returned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07-01 | Information Disclosure | GET /api/notifications returns another user's notifications | mitigate | privySessionPreHandler verifies session; userAddress in query must match authenticated user; 403 if mismatch |
| T-02-07-02 | Tampering | POST /mark-read marks another user's notifications | mitigate | WHERE clause includes userAddress = authenticatedUser; cannot mark other users' rows |
| T-02-07-03 | Denial of Service | Fan-out worker subgraph query times out on high-follower call | mitigate | subgraph query paginated (limit 100 per batch, retry); fan-out inserts in batches; if subgraph fails, log + skip + retry on next tick |
| T-02-07-04 | Repudiation | statusVersion not bumped after CallerExited | mitigate | statusVersion Redis key bump is in the same tick as notification insert; if Redis fails, log error; OG card falls back to v=0 (slightly stale but not wrong) |
| T-02-07-05 | Information Disclosure | Live-state route exposes RPC URL | accept | RPC URL is env-var server-side only; not returned in response; standard server-side-only pattern |
</threat_model>
