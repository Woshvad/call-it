<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User wallet→useWriteContract | Untrusted input for follow/fade/exit amounts |
| Frontend slippage compute→Contract | minSharesOut computed client-side; contract is authoritative; never trust only client value |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08-01 | Tampering | Frontend minSharesOut bypass | accept | Contract enforces SlippageExceeded on-chain regardless of what frontend sends; contract is the safety net |
| T-02-08-02 | Tampering | CallerExit modal skipped (direct tx) | accept | Contract enforces 24h lock (CallerExitLocked) and penalty math regardless; modal is UX friction only |
| T-02-08-03 | Information Disclosure | Wallet address shown on shareable receipt | mitigate | Live Receipt header shows handle + rep only (no wallet address per §15.7/AUTH-44); address used only for contract reads not displayed |
| T-02-08-04 | Repudiation | Status stale on page (CALLER EXITED not shown immediately) | mitigate | useReadContracts 5s refetch; optimistic state update on own tx submission; acceptable for v1 |
</threat_model>
