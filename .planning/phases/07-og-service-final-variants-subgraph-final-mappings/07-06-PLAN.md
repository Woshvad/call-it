## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel origin → Fly relayer | Cross-origin requests must be allowed ONLY for the explicit Vercel origin (not `*`); the allowlist is the boundary guard (D-04). |
| Operator secrets → deploy | SUBGRAPH_STUDIO_DEPLOY_KEY + Fly secrets + Vercel env are operator-held; never committed; NEXT_PUBLIC_* are public-by-design but must point at the right Sepolia endpoints. |
| Public receipt URL → Twitter/Farcaster preview | The OG image referenced by the receipt meta is fetched by Twitter's crawler; it must serve the correct variant + cache headers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06-01 | Spoofing / Elevation | CORS `*` exposing the relayer | mitigate | Explicit Vercel-origin allowlist via `CORS_ALLOWED_ORIGINS` (D-04); OPTIONS preflight smoke asserts the exact origin, not `*`. |
| T-07-06-02 | Information Disclosure | Subgraph Studio key leak to frontend | mitigate | D-27: SUBGRAPH_STUDIO_DEPLOY_KEY is an operator/relayer secret; never in NEXT_PUBLIC_*; the query key stays server-side. |
| T-07-06-03 | Tampering | config drift baked into Vercel build (Pitfall 5) | mitigate | NEXT_PUBLIC_* set before build + the post-deploy incognito smoke (OG render + X-Variant + no-CORS-error) catches a wrong subgraph/relayer URL. |
| T-07-06-04 | Tampering (public claim) | wrong/stale OG preview on the shared receipt | mitigate | Twitter Card Validator checklist (D-08) on all 5 variant URLs after deploy; ?v={statusVersion} cache-buster on the receipt meta. |
