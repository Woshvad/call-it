# Phase 7 — UI Design Contract

> Visual and interaction contract for Phase 7: OG service final variants + Subgraph final mappings.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is finalization of an ESTABLISHED design system, not a greenfield design.** All tokens
> below are detected-and-locked from the existing `@call-it/ui` neobrutalist system (spec §14.6).
> The contract's job here is to (1) lock the OG-variant real-data wiring + readability gates, and
> (2) prescribe the Profile Overview tab, Leaderboard page, and Quote Composer using the existing
> primitives. Executor MUST reuse `@call-it/ui` primitives — do NOT introduce new tokens.

## Spacing Scale

Declared values (all multiples of 4). Source: existing component usage (Tailwind default 4px unit;
`gap-2`/`gap-4`/`gap-6`, `p-6`) + neobrutalist border/shadow tokens in `packages/ui/src/tokens/spacing.ts`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label/value pairs (`gap-1` = 4px in stat blocks) |
| sm | 8px | Compact element spacing, badge gaps (`gap-2`) |
| md | 16px | Default element spacing, filter-chip gaps (`gap-4`) |
| lg | 24px | Card inner padding (`p-6`), stat-row gaps (`gap-6`) |
| xl | 32px | Section gaps between Profile tabs / Leaderboard blocks |
| 2xl | 48px | Major section breaks (Hero card → table) |
| 3xl | 64px | Page-level top/bottom rhythm |

**Neobrutalist structural tokens (LOCKED — `packages/ui/src/tokens/spacing.ts`):**

| Token | Value | Usage |
|-------|-------|-------|
| Border width | 3px (`border-3`) default, 4px (`border-4`) for emphasis | Card/Tag/Stamp edges, 4px corner brackets |
| Shadow offset (sm) | `2px 2px 0 0` | Tags, chips, small interactive elements |
| Shadow offset (md) | `4px 4px 0 0` | Buttons, secondary cards |
| Shadow offset (lg) | `6px 6px 0 0` | Primary cards (`Card` default = `shadow-[6px_6px_0_0_#E8F542]`) |
| Shadow color | `#000000` (black) default; `#E8F542` (accent) for highlighted `Card` | Hard offset shadows — never soft/blurred |

Exceptions:
- **OG card variants render at 1200×630** (Satori canvas) — spacing inside OG JSX uses absolute px,
  not the Tailwind scale, but stays on 4px multiples. Flexbox only (`display: grid` is BANNED — Pitfall 15).
- **Outcome word in OG cards must remain legible at a 200px-wide social thumbnail** (SC1 QA gate) —
  this dictates a large display size in the card (≥ 64px at 1200px canvas) regardless of the page scale.

## Typography

Font roles are LOCKED (`packages/ui/src/tokens/typography.ts` + `tailwind.preset.ts`). Three families,
each with a fixed role. Weight discipline: **2 weights total** (regular 400 + bold 700) — neobrutalist
treatment uses bold heavily; do not introduce 500/600 intermediate weights. (Weight discipline PASSES.)

> ### ⚠️ Locked-System Typography Exemption (recorded, deliberate — NOT visual drift)
>
> The on-page type scale below declares **6 distinct font sizes** (12 / 14 / 16 / 18 / 20 / 28px),
> which exceeds the GSD mechanical 4-size maximum. This is a **documented, recorded exemption**, not a
> redesign target, for the following reasons:
>
> 1. **The scale is LOCKED and inherited, not newly proposed.** These six sizes are the *already-shipped*
>    tokens of the `@call-it/ui` neobrutalist design system (`packages/ui/src/tokens/typography.ts` +
>    `tailwind.preset.ts`) and spec §14.6. They were **detected from the shipped tokens, not authored by
>    this phase.** Phase 7 is the *finalization* of live, shipped UI; collapsing the type scale would mean
>    redesigning production surfaces — explicitly out of scope and the wrong move.
> 2. **Each of the 6 sizes serves a distinct, already-implemented role** (justified per-size in the table
>    below). None is redundant; removing any would change the appearance of live primitives (`ProfileHeader`,
>    `CallCard`, `Receipt`, `Tag`, `Stamp`).
> 3. The OG display size (≥64px, Satori canvas) is a **separate render surface** and is correctly NOT
>    counted toward the on-page scale.
>
> **Decision: the 4-size maximum is waived for this phase as a locked-system exception. The shipped token
> values MUST NOT be altered.**

### Per-size justification (each maps to a distinct, shipped role)

| # | Size | Tailwind | Family / Weight | Distinct shipped role | Where it appears across Phase-7 surfaces |
|---|------|----------|-----------------|------------------------|-------------------------------------------|
| 1 | 12px | `text-xs` | Space Grotesk / JetBrains Mono · 400 | **Uppercase stat/section labels, chip & lozenge text, metadata captions** | Profile Overview 5-stat labels (`Accuracy`/`Calibration`/`ROI`/`Contrarian hits`/`Streak`), `CATEGORY REPUTATION`/`RECENT CALLS` section labels, filter chips, Leaderboard category chips, `Tag`/`Stamp` lozenge labels |
| 2 | 14px | `text-sm` | JetBrains Mono · 700 (values) / muted handles | **Mono numeric values in dense rows + `@handle` lines** | Leaderboard table cells (rank/rep in mono bold), Profile stat values in compact blocks, `@handle` muted (`ProfileHeader`), Quote Composer parent-context metadata |
| 3 | 16px | `text-base` | Space Grotesk · 400 | **Body copy / list rows / descriptions** | `CallCard` statement body, Quote Composer text input + thread preview body, empty-state body copy, `MOST FOLLOWED BY` rows |
| 4 | 18px | `text-lg` | JetBrains Mono · 700 / Syne · 700 | **Emphasized values & sub-headings / card titles** | Larger mono values (Hero card #1 rep, headline P&L), `Card` titles in `CATEGORY REPUTATION` and `NOTABLE RECEIPTS`, Quote success sub-heading |
| 5 | 20px | `text-xl` | Syne · 700 | **Section headings / profile display name** | Profile display name (`ProfileHeader`), section headings on Profile tab, `Quote this call` / `Quote posted` headings |
| 6 | 28px | `text-3xl` | Syne · 700 | **Page / display headings** | Leaderboard `The Tape` page title, large display headings on the Profile/Leaderboard surfaces (the giant faded "01" watermark and OG outcome word scale ABOVE this on their own canvas) |

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Body | Space Grotesk (`font-body`) | 16px (`text-base`) | 400 | 1.5 | Paragraph copy, list rows, descriptions |
| Label / caption | Space Grotesk or JetBrains Mono | 12px (`text-xs`) | 400 | 1.4 | Stat captions ("Total Calls"), chip labels, metadata |
| Mono value | JetBrains Mono (`font-mono`) | 14px (`text-sm`) / 18px (`text-lg`) | 700 | 1.3 | Numbers: accuracy %, ROI, streak, leaderboard rep, $ amounts |
| Heading | Syne (`font-display`) | 20px (`text-xl`) | 700 | 1.2 | Card titles, section headings, profile display name |
| Display | Syne (`font-display`) | 28px+ (`text-3xl`) / OG ≥64px | 700 | 1.1 | Leaderboard "01" watermark, "The Tape" title, OG outcome word |

**Rules:**
- Numeric/financial values ALWAYS render in JetBrains Mono bold (established in `ProfileHeader` stats,
  `Receipt`). Accuracy, Calibration, ROI, Contrarian hits, Streak, leaderboard rep — all mono.
- Headings/titles/outcome words ALWAYS Syne bold.
- Handles render `@handle` in mono muted (`font-mono text-sm text-brand-muted`) per `ProfileHeader`.
- Heading line-height 1.2; body 1.5 (matches existing components).
- **Type scale (6 sizes) is LOCKED — see the Locked-System Typography Exemption above. Do not collapse
  or alter the shipped token values; the 4-size maximum is waived for this finalization phase.**

## Color

LOCKED palette — `packages/ui/src/tokens/colors.ts` + `tailwind.preset.ts` (spec §14.6). The 60/30/10
split maps directly to the existing neobrutalist tokens. Accent (`#E8F542`) is scarce by design.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090E` (`brand-bg`) | Page background, all primary surfaces |
| Secondary (30%) | `#18181B` (`brand-surface`) | Cards, table rows, nav, panels, chips |
| Border / structure | `#27272A` (`brand-border`) + `#FFFFFF` (default Card border) | 3px hard borders, dividers |
| Text | `#FFFFFF` (`brand-text`) primary; `#A1A1AA` (`brand-muted`) secondary/captions | Body, labels, metadata |
| Accent (10%) | `#E8F542` (`brand-accent`) | See reserved-for list below |
| Destructive | `#EF4444` (`outcome-loss`) | Loss states, destructive confirmations only |

**Accent (`#E8F542`) reserved for — EXACT list (never "all interactive elements"):**
1. Primary CTA buttons (e.g. "Quote this call", "Share receipt").
2. Conviction / follow-fade progress bar fill (follow side) in OG cards and `ConvictionBar`.
3. The viewer's own highlighted row in the Leaderboard table.
4. The Leaderboard #1 Hero card border + the giant faded "01" watermark (rendered at low opacity).
5. Card `accent` variant border + the hard offset shadow (`shadow-[6px_6px_0_0_#E8F542]`).
6. `VerifiedBadge` (reuses `Tag intent=warning` brand-accent).
7. Active filter-chip / active tab indicator (7d/30d/All-time toggle, category chips, profile tabs).
8. The `CONTRARIAN HIT` and `FADED CORRECTLY` outcome words (§14.1 — both `#E8F542`).

**Outcome-word colors — LOCKED §14.1 (explicit hex in `lib/outcome-word.ts`, NOT the Stamp token map):**

| LOUD AND WRONG | `#F87171` | loss red (brighter than `outcome-loss #EF4444` — use §14.1 value) |
| CONTRARIAN HIT | `#E8F542` | accent — NOT purple `#A855F7`; lozenge label "CONTRARIAN" |

## OG Card Variant Contract (SC1 — finalization, not redesign)

All 5 variants are built; this phase wires real data and adds enforcement gates. Contract:

| Variant | Route | Real-data wiring (D-03) |
|---------|-------|--------------------------|
| Live | `og/[callId]` (`buildLiveCard`) | follow%/fade% bar + time-left already live; wire market statement from subgraph `Call` entity field (replaces `Call #N` stub) |
| Settled | `og/[callId]` (`buildSettledCard`) | outcome word via `getOutcomeWordResult`; wire P&L / REP CHANGE / FINAL / TARGET from SettlementManager settlement events via subgraph (replaces `—` stubs) |
| DuelSettled | `og/duel/[challengeId]` | wire settled fields from subgraph (replaces stubs) |
| CallerExited | `og/[callId]` (`buildCallerExitedCard`) | wire statement + exit data from subgraph |
| Fallback | `og/fallback` / `api/og/fallback` | Phase-0, working — no change; rendered on any throw (SHARE-10) |

**LOCKED OG invariants (enforced by SC1 gates):**
- `export const runtime = 'nodejs'` first line of every OG route.
- **Flexbox only. `display: grid` is BANNED.** SC1 adds a custom eslint rule rejecting `display: grid`
  (and `display: 'grid'`) in OG source files. Pitfall 15 — Satori does not support grid.
- Outcome word must be legible at a **200px-wide social thumbnail** — SC1 adds a Playwright visual-regression
  gate at 200px viewport for all 5 variants; the §14.1 outcome color + Syne bold display size satisfy it.
- All 5 variants must pass the **Twitter Card Validator pre-flight** (SC1).
- `Cache-Control: max-age=60, stale-while-revalidate=300` + `X-Variant` header on every OG response.
- Receipt page server-renders OpenGraph meta with `?v={statusVersion}` cache-buster; receipt/profile/duel/
  leaderboard are PUBLIC (no auth) — middleware carve-out already in place.

## Layout Prescriptions (net-new / built-out surfaces)

Reuse `@call-it/ui` primitives; all flexbox; neobrutalist treatment (3px borders, hard offset shadows,
4px corner brackets). No grid anywhere (consistency with OG + Pitfall 15).

### Profile Overview tab (§15.4)
- **5-stat row:** horizontal flex row of 5 stat blocks (label `text-xs brand-muted` + value mono bold).
  Stats: Accuracy / Calibration / ROI / Contrarian hits / Streak. Wraps to flex-wrap on narrow widths
  (full mobile responsive is Phase 9 — desktop-first here).
- **CATEGORY REPUTATION grid:** 3 `Card`s in a flex row (not CSS grid). Each card = category name +
  mono rep value + small bar.
- **RECENT CALLS:** `CallCard` list with filter chips (`All`/`Open`/`Settled`); active chip uses accent.
- **MOST FOLLOWED BY:** compact avatar+handle list (reuse `ProfileHeader` avatar treatment).
- **NOTABLE RECEIPTS:** trophy `Card`s using `Stamp` + outcome word render.

### Leaderboard page (§15.6)
- **Title block:** "The Tape" (Syne display) + "Top of book" (muted body subtitle).
- **Time toggle:** 7D / 30D / ALL-TIME segmented control; active = accent.
- **Category chips:** All / Majors / DeFi / Other; active = accent.
- **#1 Hero card:** accent-bordered `Card` with a giant faded "01" Syne watermark (low-opacity accent,
  behind content) + the #1 caller's handle/stats.
- **Table:** rows on `brand-surface`; **the viewer's own row is highlighted with accent** (left border
  or background tint). Rank in mono bold, rep in mono bold, handle in mono.

### Quote Composer + success (§15.10)
- **Parent context card:** the quoted call shown as a read-only `Card` (statement + caller + status).
- **Composer:** text input + stance + `Post quote` (accent) / `Cancel` (ghost).
- **Thread preview:** parent card + the user's quote stacked vertically (thread visual).
- **Success screen:** "Quote posted" heading + the rendered thread + `Share receipt` CTA.
