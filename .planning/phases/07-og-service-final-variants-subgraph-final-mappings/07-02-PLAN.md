<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend /new → POST /api/calls/criteria | Caller-supplied statement prose crosses into the relayer store; untrusted, must be length-capped. |
| Relayer DB → live-state marketLine → OG/receipt | The stored statement is rendered into a public OG card + receipt; must be treated as text (Satori renders text, not HTML) and escaped if the receipt page renders HTML. |
| On-chain CallCreated → subgraph mapping | The event carries no string; the mapping can only template from numerics — never assume prose is present. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering | statement string in OG/receipt render | mitigate | Length-cap on persist (V5); OG truncates at 77/87 chars; receipt page escapes if rendered as HTML; subgraph templated fallback guarantees a safe default. |
| T-07-02-02 | Denial of Service | call creation blocked by DB write | mitigate | Statement persist is in the fail-safe non-fatal try/log/continue block — a DB outage logs + continues, never fails `createCall`. |
| T-07-02-03 | Information Disclosure | Subgraph Studio query key | mitigate | D-27: subgraph reads stay relayer-side; this plan adds no NEXT_PUBLIC_* subgraph key. |
| T-07-02-SC | Tampering | npm installs | mitigate | No new packages — drizzle-orm 0.45.2 / drizzle-kit 0.31.10 + graph-cli 0.98.1 already pinned; slopcheck N/A. |
</threat_model>
