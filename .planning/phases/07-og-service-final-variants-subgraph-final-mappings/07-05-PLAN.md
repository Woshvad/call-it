## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Web pages → Fly relayer (/api/*) | Pages fetch the relayer client-side; the cross-origin allowlist (D-04) is set at deploy (Plan 07-06). Studio subgraph key stays server-side (D-27). |
| Quote submit → quote_stance route | Quote mutations stay Privy-session-gated (existing pattern); the public read surfaces (leaderboard/profile) carry no session. |
| Share button text → public tweet | Uses the pure share-text builder; statement is URL-encoded; no secret in the URL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05-01 | Information Disclosure | Subgraph Studio key in frontend bundle | mitigate | D-27: `getLeaderboard` reads via the relayer/subgraph-client server-side; key never in `NEXT_PUBLIC_*`. |
| T-07-05-02 | Spoofing / Elevation | CORS `*` exposing the relayer | mitigate | Explicit Vercel-origin allowlist set at deploy (D-04, Plan 07-06); pages assume the allowlist, not `*`. |
| T-07-05-03 | Tampering | untrusted statement in quote/share render | mitigate | Statement treated as text + URL-encoded in share-text; receipt/profile render escapes HTML. |
| T-07-05-SC | Tampering | npm installs | mitigate | No new packages — reuse `@call-it/ui` + existing Playwright; slopcheck N/A. |
