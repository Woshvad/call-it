Rationale: §4.3.7 protocol milestones are predominantly TVL/fees-based (e.g., "Uniswap TVL exceeds $X") which DefiLlama already handles. If on-chain validator/supply-ratio milestones are needed, the owner can call setAdapterMap(2, 8, RpcMetrics) post-deploy without redeploying.

**NFT routing clarification:** NFT calls use MarketType.PriceTarget(0) + EventSubtype.None(0). adapterMap[0][0] = Pyth(0) by default. The relayer watcher checks whether assetA encodes a Pyth feedId vs an NFT contract address and routes accordingly (pre-existing logic in settlement-watcher.ts lines 486-515). No separate adapterMap slot is needed for NFT-TWAP in this plan.
