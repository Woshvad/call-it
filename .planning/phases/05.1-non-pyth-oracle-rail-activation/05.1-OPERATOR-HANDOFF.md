## Remaining → Phase 6 soak (relayer go-live)
The relayer (`call-it-relayer-sepolia`) is `pending` and operates the non-Pyth rail during the soak. When deploying it:
1. **Fly secrets / process.env:** set `KMS_ADDRESS_NFT_TWAP`, `KMS_ADDRESS_DEFILLAMA`, `KMS_ADDRESS_SNAPSHOT_TALLY`, `KMS_ADDRESS_CEX` (above) and `GCP_LOCATION_ID=us-east1` (code defaults to us-east1, but be explicit). KMS signer addresses are read from `process.env` by the oracle adapters.
2. **GCP Secret Manager:** update `NEXT_PUBLIC_CALL_REGISTRY_ADDRESS` → `0x9E3E467e5D1F1266354444CEaC67651c7e9CACEc` (the relayer's `secret-manager.ts` fetches this from GCP Secret Manager, not fly secrets). SM/CR also come from the rebuilt `@call-it/shared` (addresses.ts already updated).
3. **IAM:** grant the relayer's runtime GCP service account `roles/cloudkms.signerVerifier` on the 5 keys (so it can sign): `gcloud kms keys add-iam-policy-binding <key> --keyring attestations --location us-east1 --project call-it-sepolia --member serviceAccount:<RELAYER_SA> --role roles/cloudkms.signerVerifier`.
4. **Backfill:** once non-Pyth test calls are seeded on the new CR, populate `apps/relayer/src/scripts/backfill-criteria.ts` `SEPOLIA_TEST_CALLS` and run it so `call_oracle_criteria` has rows for DefiLlama/RpcMetrics/CEX calls.
5. **Frontend:** point `NEXT_PUBLIC_SUBGRAPH_URL` at the new Studio version once it has synced; rebuild/redeploy frontend with updated `@call-it/shared`.
