## Deploy blocker (definitive, 2026-06-02): KMS signing keys are NOT provisioned

Cloud KMS API was enabled on project `call-it-sepolia`, but `gcloud kms keys versions get-public-key` returns **`NOT_FOUND`** for all four keys — the keyring `attestations` and keys `nft-twap` / `defillama` / `snapshot-tally` / `cex` **do not exist**. The attestation signing infra was never provisioned, so the 4 `KMS_ADDRESS_*` cannot be derived and the deploy's `attestationSigner` assertions cannot pass. **Config inconsistency to resolve first:** `.env` `GCP_LOCATION_ID=global` (path seen: `…/locations/global/keyRings/attestations/…`) vs `kms-signer.ts` comment "must be `us-east1`".

**Provisioning checklist (deliberate, security-critical — operator-owned):**
1. Decide the KMS **location** and make `.env` `GCP_LOCATION_ID` + `kms-signer.ts` consistent (`global` vs `us-east1`).
2. Create keyring `attestations` and 4 crypto keys with purpose `asymmetric-signing`, algorithm `ec-sign-secp256k1-sha256`:
   `gcloud kms keyrings create attestations --location=<loc> --project=call-it-sepolia`
   `gcloud kms keys create <nft-twap|defillama|snapshot-tally|cex|oauth-proof> --keyring=attestations --location=<loc> --purpose=asymmetric-signing --default-algorithm=ec-sign-secp256k1-sha256 --protection-level=hsm --project=call-it-sepolia`
   **NOTE:** `--protection-level=hsm` is REQUIRED — secp256k1 is HSM-only in GCP KMS; `software` fails with `INVALID_ARGUMENT` (the Phase-0 `00-02-PLAN.md` recipe is wrong on this). HSM ≈ $1–2.50/key/mo.
3. Grant the **relayer's runtime GCP service account** `roles/cloudkms.signerVerifier` on each key (so the relayer can sign; the deploy derives the same addresses from the public keys).
4. Set `GCP_KEY_VERSION_*` to `1` (or the active version). Then the deploy can derive `KMS_ADDRESS_*` (type1=NFT_TWAP, 2&3=DEFILLAMA, 4&5=SNAPSHOT_TALLY, 6=CEX) and proceed.

Derivation logic for the 4 addresses: `keccak256(uncompressed_pubkey_XY)[12:]` (see `apps/relayer/src/lib/kms-signer.ts::verifyKmsAddress`). Once the keys exist, the deploy + Plan 05 can be run end-to-end (the relayer also needs these keys to sign live).
