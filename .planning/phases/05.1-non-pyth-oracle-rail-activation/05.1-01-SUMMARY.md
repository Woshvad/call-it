## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Em-dash characters in Solidity string literals**
- **Found during:** Task 2 (first forge test run)
- **Issue:** The initial AdapterMapTest.sol had two em-dash characters (U+2014) inside string literals (`assertEq`/`assertTrue` messages). Solidity rejects non-ASCII in string literals without the `unicode""` prefix.
- **Fix:** Replaced both em-dashes with `--` (double-hyphen) in the two affected string literals. Comment lines and NatDoc lines were left unchanged (comments accept Unicode).
- **Files modified:** `packages/contracts/test/AdapterMapTest.sol`
- **Commit:** 4d26a00 (inline fix before commit)

**2. [Plan note] testNonPythAttestationRejectedWithoutAdapterMap implementation**
- The plan noted an optional path: "if setting up a full call is too complex within the test scope, use vm.expectRevert and call _checkAdapterBinding via a test-harness wrapper, OR document that the attestation rejection is already covered by SettlementAttestationTest.sol and use an assertEq on the raw adapterMap return value instead."
