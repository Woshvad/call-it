# Phase 05.1 Plan 03: Non-Pyth Adapter Rewire + Settlement Watcher Fix Summary

**All 6 non-Pyth adapters rewired to unified oracle-attestation.ts (CallIt-Oracle EIP-712 domain, chainId from env); settlement-watcher now dispatches on on-chain adapterMap(marketType, eventSubtype) read; CEX Pitfall 6 wired; 19-field canonical ABI restored**

## Tasks Completed

### Task 1: Fix settlement-watcher.ts (commit `495c26d`)

**Files modified:** `apps/relayer/src/workers/settlement-watcher.ts`

Changes:
- Replaced 14-field fabricated `CALL_REGISTRY_ABI` (wrong field names: `assetA_feedId`, `assetB_feedId`; missing 5 fields) with canonical 19-field struct matching `ICallRegistry.sol:130-157`
- Added `ADAPTER_MAP_ABI` constant (derived from `SettlementManager.sol:119-120` public mapping getter)
- Added `SUBMIT_ATTESTATION_ABI` import from `oracle-attestation.ts`; removed 5 inline `SM_SUBMIT_ABI_*` duplicate constants
- Replaced `const adapterType: OracleAdapter = call.marketType as OracleAdapter` with `publicClient.readContract('adapterMap', [call.marketType, call.eventSubtype])` on-chain read
- Renamed all `call.assetA_feedId` → `call.assetA` and `call.assetB_feedId` → `call.assetB`
- Wired `walletClient` into `CexAdapter` config so `scrapeAndAttest` can call `SM.submitAttestation` internally
- Updated NftTwap case to use `walletClient.writeContract` + `SUBMIT_ATTESTATION_ABI` instead of `submitNftFloor` call

### Task 2a: Rewire governance adapters (commit `826d7f0`)

**Files modified:** `snapshot-adapter.ts`, `tally-adapter.ts`

Changes for both:
- Removed `SNAPSHOT_ATTESTATION_TYPES` / `TALLY_ATTESTATION_TYPES` legacy EIP-712 type blocks
- Removed per-adapter domain blocks (`name: 'CallIt-SnapshotTally'`, `chainId: 42161n`)
- Replaced custom `account.signTypedData(...)` + `encodeAbiParameters(...)` with `signOracleAttestation({ oracleType: OracleType.Snapshot (4) / OracleType.Tally (5), priceDelta: 0n, chainId: BigInt(process.env.CHAIN_ID!) })`
- `chainId` from `process.env.CHAIN_ID` -- never hardcoded (T-05.1-03-01)
- Return type updated to use `result.fields.outcome`, `result.fields.timestamp`, `result.signature`, `result.attestationData`

### Task 2b: Rewire value adapters (commit `3bcfbeb`)

**Files modified:** `defillama-adapter.ts`, `rpc-metrics-adapter.ts`, `nft-twap-adapter.ts`, `cex/cex-adapter.ts`

Changes for all 4:
- Removed `DEFILLAMA_ATTESTATION_TYPES`, `RPC_METRICS_ATTESTATION_TYPES`, `NFT_TWAP_ATTESTATION_TYPES`, `CEX_ATTESTATION_TYPES` legacy blocks
- Removed per-adapter domain blocks (all had `chainId: 42161n` and per-adapter names)
- Replaced with `signOracleAttestation` using correct `OracleType.{DefiLlama=2, RpcMetrics=3, NftTwap=1, CexScraper=6}`
- Added `targetValue` parameter to all value adapter `fetchAndAttest` methods
- All 3 value adapters (DefiLlama, RpcMetrics, NftTwap): return `{ ambiguous: true }` if `targetValue === 0n` (T-05.1-03-07); pass `resolveValueOutcome(observedValue, targetValue)` to determine outcome + priceDelta
- `submitNftFloor` standalone function updated to use `SUBMIT_ATTESTATION_ABI` (Pitfall 5 fix)
- `CexAdapter.scrapeAndAttest`: now calls `SM.submitAttestation` internally after signing (Pitfall 6 fix); `CexScrapeOutcome` type upgraded to `{ status: 'found'; attestationData; signature } | { status: 'not_found' } | { status: 'ambiguous' }`
- `createCexAdapter` updated to accept optional `walletClient`
