<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Adapter → signOracleAttestation | EIP-712 signing boundary; chainId must match the deployed contract's block.chainid |
| Relayer → SettlementManager.submitAttestation | On-chain ECDSA.recover checks signature against attestationSigner[oracleType]; any domain mismatch = InvalidAttestation revert |
| settlement-watcher → adapterMap on-chain read | Dispatch is now chain-authoritative; compromise of the adapterMap owner key could reroute attestations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.1-03-01 | Spoofing | chainId hardcoded to 42161 in legacy adapters | mitigate | Acceptance criteria explicitly requires grep for 42161n returning empty in all 6 adapter files; process.env.CHAIN_ID must be set correctly in relayer env (421614 on Sepolia, 42161 on mainnet) |
| T-05.1-03-02 | Spoofing | Wrong domain name (CallIt-SnapshotTally vs CallIt-Oracle) | mitigate | Acceptance criteria requires grep for CallIt-SnapshotTally and CallIt-DefiLlama returning empty; oracle-attestation.test.ts 12/12 validates the correct domain is used |
| T-05.1-03-03 | Spoofing | Forged attestation with oracleType != adapterMap configured type | mitigate | _checkAdapterBinding at SettlementManager.sol:591 is the on-chain binding; this plan ensures each adapter passes the correct oracleType enum value (4 for Snapshot, 5 for Tally, etc.) that matches the adapterMap configured in Plan 02 |
| T-05.1-03-04 | Tampering | settlement-watcher reading stale adapterMap after contract redeploy | mitigate | Watcher uses live on-chain readContract call per dispatch cycle; as long as SETTLEMENT_MANAGER address env var is updated to the new SM address (Plan 05 Task 1), the read will always query the current contract |
| T-05.1-03-05 | Elevation of Privilege | CEX adapter previously logged but never submitted — could be exploited if stub was silently relied upon | mitigate | submitAttestation is now wired in cex-adapter scrapeAndAttest; the log-only path is removed; no path remains where attestation appears signed but is never submitted |
