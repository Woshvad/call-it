<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → raiseDispute() | USDC bond required; permissionless; $5 USDC transferred on-chain |
| Owner wallet → resolveDispute() | Client-side isOwner check + on-chain onlyOwner revert if bypassed |
| IPFS evidence → Pinata | Pinata Phase 0 pipeline; CID is content-addressed; hash stored on-chain |
| /api/disputes/resolve | Privy auth on relayer (if relayer-mediated); else direct contract call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08-01 | Elevation | Non-owner calls resolveDispute | mitigate | On-chain onlyOwner revert is the primary defense; client-side isOwner gate is convenience; UI note: "Owner wallet required" |
| T-04-08-02 | Repudiation | Owner resolves without preview | mitigate | Reversal preview REQUIRED before confirm — if preview fetch fails, confirm is DISABLED (D-07 compliance) |
| T-04-08-03 | DoS | Dispute spam (many fake disputes) | mitigate | $5 USDC bond is the anti-spam mechanism; MAX_COUNTER_CLAIMS=3 prevents pile-on; SETTLE-30 enforces on-chain |
| T-04-08-04 | Repudiation | forceSettle without 24h commitment | mitigate | Public /disputes/ log shows owner commitment deadline publicly (Pitfall 6); Telegram dispute_raised alert fires on new dispute |
| T-04-08-05 | Information Disclosure | Wallet address in dispute UI | mitigate | AUTH-44: all dispute surfaces show @handle only; addresses never rendered in copy or UI |
</threat_model>
