<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Relayer → Subgraph GraphQL | Relayer fetches from subgraph — untrusted external data; must validate shape |
| Relayer → live RPC reads | ChallengeEscrow.getChallenge RPC call — trusted but can timeout |
| Client → /api/duels/:id/live-state | Unauthenticated read; challengeId param from URL |
| BullMQ worker → Postgres | Worker writes trending_duels; must be idempotent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-05-01 | Tampering | challengeId param injection in duel-live-state | mitigate | Parse to BigInt with try/catch; return 400 on invalid; cacheKey uses BigInt.toString() (no string interpolation of raw param) |
| T-3-05-02 | Denial of Service | RPC timeout in duel-live-state | mitigate | try/catch wraps readContracts; on error log + return 503; Redis cache serves last-known value on next request within TTL |
| T-3-05-03 | Denial of Service | Trending worker crash loop | mitigate | tick() is wrapped in try/catch — any error increments errors counter and logs but does NOT throw; setInterval continues |
| T-3-05-04 | Tampering | Subgraph data poisoning (malformed event data) | mitigate | Validate pot and backerCount as BigInt before threshold comparison; NaN/undefined returns skip the upsert; log warning |
| T-3-05-05 | Elevation of Privilege | Unauthenticated /api/duels modification | accept | Routes are read-only GET endpoints; no auth gate needed per spec §18.1 (public reads); no mutation exposed |
| T-3-05-06 | Information Disclosure | RPC URL in server logs | mitigate | RPC URL stored in ARBITRUM_SEPOLIA_RPC_URL (server-side env only, never NEXT_PUBLIC_*); not logged; consistent with live-state.ts pattern |
</threat_model>
