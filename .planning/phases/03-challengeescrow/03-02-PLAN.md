<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| USDC transfer in→contract | proposeChallenge/acceptChallenge: challenger/caller safeTransferFrom. CEI must hold. |
| USDC transfer out→user | claimDuelPayout/claimOverage/rejectChallenge/claimRefund: push to user. reentrancy vector. |
| settleDuel authorization | Only SettlementManager (address(0) at deploy) may call settleDuel. Any caller can forge if not guarded. |
| _pushOverage push path | Sending USDC to potentially malicious wallet during settleDuel. Must not revert settleDuel on failure. |

| T-3-02-02 | Spoofing | SelfChallenge rep farming | mitigate | proposeChallenge gate: `if (msg.sender == c.caller) revert SelfChallenge()` checked second (after openToChallenges); cannot be bypassed at contract level |
| T-3-02-03 | Elevation of Privilege | settleDuel unauthorized call | mitigate | onlySettlementManager modifier; settlementManager = address(0) at deploy; only owner can rotatevia setSettlementManager |
| T-3-02-04 | Denial of Service | _pushOverage griefing | mitigate | IERC20.transfer bool return (not safeTransfer); on false: record UnclaimedOverage; settleDuel always completes; testOveragePushFail verifies |

| T-3-02-08 | Denial of Service | Paused contract blocking claim/exit | mitigate | §10.3 pause carve-out: claimDuelPayout and claimOverage NOT guarded by whenNotPaused; comment present on both functions |
