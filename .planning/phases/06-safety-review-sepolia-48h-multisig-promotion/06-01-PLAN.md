## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Solidity source → deployed bytecode | resolveUsdc() correctness is a compilation + deployment trust boundary — if the source is correct but the wrong bytecode is deployed, the invariant fails |
| CI grep guard → merged source | The guard only fires on CI — a dev could locally bypass it; the Foundry fork tests are the deeper backstop |
| Security review checkpoint → redeploy trust | The deploy proceeds only after human confirmation of the ADR-0001 checklist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Tampering | USDC.sol resolveUsdc() | mitigate | Dual-branch structure: mainnet path is first and unconditional on chainid 42161; Sepolia path second; revert catch-all. Foundry USDC.t.sol tests all three paths. |
| T-06-01-02 | Tampering | ChallengeEscrow/SettlementManager constructor | mitigate | require(_usdc == resolveUsdc()) at construction time; _usdc is immutable after that point — no setter exposed. |
| T-06-01-03 | Spoofing | CI grep guard bypass | accept | A developer could add a USDC address outside the allowlist locally and not push; accepted because the Foundry constructor asserts are the on-chain backstop, not CI alone. |
| T-06-01-04 | Elevation of Privilege | Wrong chain deploy | mitigate | resolveUsdc() reverts on unsupported chainid — any accidental deploy to e.g. Ethereum mainnet (chainid 1) fails at construction. |
| T-06-01-05 | Information Disclosure | USDC_ARB_SEPOLIA constant in source | accept | Circle Sepolia USDC is a public address; no secret. |
| T-06-01-06 | Denial of Service | resolveUsdc() revert on Sepolia upgrade tools (e.g. foundry test with default chainid) | mitigate | Test suite uses vm.chainId() cheat codes to set chainid before any contract deployment; USDC.t.sol covers all three chainid branches. |
