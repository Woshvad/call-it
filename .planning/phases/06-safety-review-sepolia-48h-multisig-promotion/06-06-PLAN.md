<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Deployer key → Safe | transferOwnership initiates the transfer; until acceptOwnership fires, deployer still holds owner() — this 2-step window is intentional |
| Safe threshold → execution | 2-of-3 threshold requires 2 signers; if signer 2 is unavailable, execution stalls. Solo 2-of-3 means operator holds all 3 keys, so stall = operator problem |
| Arbitrum One production Safe → Phase 7 mainnet ownership | The production Safe is funded and ready but not yet the owner of mainnet contracts — that is Phase 7; TransferOwnershipToSafe.s.sol address constants must be updated to mainnet values before Phase-7 use |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06-01 | Elevation of Privilege | Ownable2Step pendingOwner window | mitigate | transferOwnership only sets pendingOwner; owner() unchanged until Safe executes acceptOwnership. If Safe fails to call acceptOwnership, deployer retains ownership — no lock-in. |
| T-06-06-02 | Denial of Service | Safe threshold stall on acceptOwnership | accept | Solo 2-of-3: operator holds all 3 keys — stall is self-inflicted and recoverable. Multi-party multisig is v1.1 scope per deferred ideas. |
| T-06-06-03 | Elevation of Privilege | Deployer retains access to ProxyAdmin after plain Ownable transfer | mitigate | ProxyAdmin.transferOwnership() is immediate (plain Ownable) — deployer loses ProxyAdmin access in the same tx. Verify with cast call PROXY_ADMIN "owner()" == Safe immediately after transferOwnership script. |
| T-06-06-04 | Spoofing | SAFE_ADDRESS env var pointing to wrong Safe | mitigate | deploy-safe.ts logs the deployed Safe address; operator sets SAFE_ADDRESS from that output. TransferOwnershipToSafe.s.sol reads SAFE_ADDRESS from env and asserts pendingOwner == SAFE_ADDRESS post-broadcast — mismatch exits non-zero. |
| T-06-06-05 | Tampering | Safe-gated upgrade sends wrong impl address | mitigate | Destruction drill (Plan 05) already proved the upgrade path works. Phase 6 Safe-gated upgrade uses the same ProxyAdmin.upgradeAndCall call via Safe UI — same codepath, different signing mechanism. |
| T-06-06-06 | Tampering | TransferOwnershipToSafe.s.sol used on mainnet with stale Sepolia addresses | mitigate | Phase-7 address update warning added as block comment + documented in SUMMARY; the script's Sepolia constants would fail at the post-broadcast require() checks when running against mainnet (wrong contract addresses = wrong pending owners). |
</threat_model>
