<objective>
Vertical slice B: the first-time signed-in user completes a 4-screen onboarding flow (Handle → Connect Socials → Follow-graph opt-in conditional → Tagline commitment) with Privy custody disclosure on Screen 1, a Coinbase Onramp + direct USDC transfer funding path before Tagline, an export prompt that fires when balance hits ≥$50, and server-side persistence of onboarding state in Fly Postgres so closing the browser mid-onboarding resumes at the same step. Relayer endpoint `POST /api/onboarding/advance` is gated by a new privy-session preHandler that becomes the canonical session-auth middleware for every subsequent user-facing relayer route in Plans 07/08/09.

Purpose: Critical-path step 2 (Fund) lands here. Onboarding is the only place where AUTH-22 custody disclosure has a guaranteed render moment — if it slides into a deferred "Settings page only" path, users may publish calls without ever seeing it. The state row pattern (D-32) is the resume defense (Pitfall B — onboarding row drift on resume).

Output: A 4 (or 5 with Fund as a step) screen onboarding flow that persists state to Postgres, surfaces all three spec-locked microcopy items (AUTH-21 commitment, AUTH-22 custody, AUTH-38 OAuth custody), wires the $50 export prompt with USDC balance polling, lands Coinbase Onramp as a hosted popup, and ships a privy-session relayer preHandler that downstream plans reuse.
</objective>
