<objective>
Ship the two Phase 1 Solidity contracts (`CallRegistry`, `ProfileRegistry`) compiled under `=0.8.30` with OZ 5.6.1 primitives, the shared `DuplicateHashLib`, a complete Foundry test surface (unit + fuzz + parity + safety) backed by a `gate-matrix.json` fixture consumed by both Solidity and Plan 03's TypeScript parity test, the `DeployPhase1.s.sol` deploy script run against Arbitrum Sepolia, and the resulting deployed addresses recorded in `packages/shared/src/constants/addresses.ts`. This is the Wave 0 contract foundation — every downstream slice (paymaster handoff, New Call form, feed, profile, parity diff CI gate) depends on these contracts existing on-chain with correct ABI.

Purpose: Without `CallRegistry.sol` shipping all 10 gates per RESEARCH Pattern 7, the contract↔preflight parity test (D-29) has nothing to assert against and the Phase 1 anti-drift guarantee evaporates. Without the `gate-matrix.json` fixture, the Plan 03 Vitest parity test cannot exist and the build cannot fail-loud on schema divergence.

Output: Two non-upgradeable Solidity contracts deployed to Arbitrum Sepolia; a Foundry test suite at green with ≥1000-run fuzz on the gate matrix; a shared JSON fixture with ≥28 cases driving both languages' parity tests; deploy addresses appended to `packages/shared/src/constants/addresses.ts`.
</objective>
