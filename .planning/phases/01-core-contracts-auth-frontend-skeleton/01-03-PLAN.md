<objective>
Land the Phase 1 anti-drift backbone: the Zod schemas and TypeScript helpers that mirror the Solidity gate logic, the cross-language parity test infrastructure, and the CI workflow that fails-loud on any divergence. This plan turns the abstract D-29 mandate into runnable code ‚Äî a fixture-driven parity matrix consumed by both Foundry and Vitest, plus a `parity-diff` script that compares their outputs case-by-case. After this plan, no future change to `CallRegistry.sol` can land without an equivalent change to `call-gates.ts` (or the build breaks), and vice versa.

Purpose: D-29 is the Phase 1 invariant guard analogous to Phase 0's grep guards. Plan 02 shipped the Solidity side of the parity; Plan 03 ships the TypeScript side and the diff comparator that elevates "two languages happen to agree" into "two languages MUST agree or CI rejects the PR." Plan 08 (New Call form) consumes these schemas via the RHF zodResolver; without them in place, Plan 08 has nothing to import.

Output: A `@call-it/shared` package with new `validation/call-gates.ts`, `types/call.ts`, and `hashing/duplicate-hash.ts` modules; two Vitest parity tests reading the same JSON fixture as Plan 02's Foundry test; a `parity-diff` script wired into `pnpm run` and into the new `phase-1-gates.yml` workflow; and a clean barrel re-export so Plan 08 can `import { createCallSchema, computeDuplicateHash } from '@call-it/shared'`.
</objective>
