<objective>
Vertical slice B: make a Call It receipt actionable from a Farcaster cast. After this slice, `POST /api/frame/tx/:callId` returns a well-formed `eth_sendTransaction` wire object whose button set is chosen by live call status (D-02/D-06) and whose calldata is built server-side with viem from the existing ABIs + pinned Sepolia addresses. This is the independently-demonstrable "tx wire is correct on testnet" deliverable (SC2) — unit-validatable end-to-end via encode/decode round-trip, with the live in-Warpcast broadcast correctly deferred to Phase 10 (D-01; Arbitrum Sepolia 421614 is not in Warpcast's chainList).

Purpose: SC2 is the action half of SHARE-19. Of the four button actions, only the three directly-constructible one-tap actions (Live Follow, Live Fade, Challenge) return tx calldata; the two non-constructible actions (settled social Follow, Quote) route to the deep-link per D-06a. This is the security-sensitive surface (a public no-auth route that constructs transaction calldata from a path param) — the threat model below is the core of the plan.
Output: 1 promoted ABI file (+ barrel export), 1 new public Frame tx route.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Farcaster client (untrusted POST signature packet) -> /api/frame/tx/:callId | Public, no-auth route that constructs transaction calldata from the path param + status; the request body is untrusted input |
| Frame tx route -> Arbitrum Sepolia contracts | The route only BUILDS calldata + `to`; the user's wallet (via Warpcast) signs/broadcasts — no server signer |
| Frame tx route -> relayer /api/calls/:id/live-state | Status read to pick the button set; fail-safe to 'Live' |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03-01 | Tampering | tx `to`/`data` could target an attacker contract | mitigate | `to` is ALWAYS a pinned `@call-it/shared` Sepolia address (`FOLLOW_FADE_MARKET_ARBITRUM_SEPOLIA` / `CHALLENGE_ESCROW_ARBITRUM_SEPOLIA`), NEVER from request params; `data` is built by `encodeFunctionData` against a const ABI with a fixed functionName — no arbitrary fn/args. Acceptance asserts the `to:` grep + decode round-trip. |
| T-08-03-02 | Tampering / DoS | malicious callId (0 / non-numeric / overflow) | mitigate | `BigInt(callIdStr)` + reject `0n` inside try/catch -> 400 BEFORE any calldata build (replicates `/og/[callId]/route.ts:739-750`). Acceptance asserts `'0'`/`'abc'` -> 400, never encodes. |
| T-08-03-03 | Spoofing | SSRF/abuse via embed imageUrl or status fetch | mitigate | The route does not fetch a user-supplied URL; the only outbound fetch is the relayer `/live-state` at an env-pinned `RELAYER_URL` with the callId path-segment validated as a BigInt. The embed `imageUrl` (Plan 02) is origin-locked to `NEXT_PUBLIC_OG_BASE_URL`. |
| T-08-03-04 | Tampering | one-tap tx with a wrong/larger amount drains > $1 | mitigate | Follow/Fade amount is HARDCODED `MIN_POSITION_USDC` (1_000_000n); never read from the Frame POST body. Larger stakes go through the deep-link, not the Frame (D-07). Acceptance asserts the decoded args === `1000000n`. |
| T-08-03-05 | Elevation of Privilege | a server-side signer in apps/web | mitigate | The route returns calldata ONLY; the user's Warpcast wallet signs + broadcasts. No private key in apps/web. Acceptance: no signer import; route returns wire JSON, not a tx hash. |
| T-08-03-06 | Spoofing | deep-link targets point off-origin | mitigate | Settled Follow/Quote deep-links are built from `NEXT_PUBLIC_OG_BASE_URL` + a fixed path (`/profile/:caller`, `/new?parent=:id`) — origin-locked, no open redirect. |
| T-08-03-07 | Tampering (functional revert) | one-tap tx reverts on missing USDC allowance (Pitfall 3) | accept (documented) | A single Frame tx cannot approve+act; the one-tap path assumes allowance and routes allowance-less users to the in-app deep-link (which runs the two-step approve->act). Stated in code; not a security hole (the tx simply reverts client-side, no fund loss). |
| T-08-03-SC | Tampering | npm installs | mitigate | Zero new npm deps — viem is already in-repo; no `@farcaster/*` ([ASSUMED]) package introduced (the SPA path is Phase 10). |
</threat_model>
