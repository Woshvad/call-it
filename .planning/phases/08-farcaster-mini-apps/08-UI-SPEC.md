## Scope Note (read first)

Phase 8 is predominantly **server-rendered Farcaster Mini App plumbing** (embed meta tags in `generateMetadata`, `/.well-known/farcaster.json` manifest, a Frame transaction endpoint). The Frame action buttons are **rendered by the Warpcast client, NOT by our CSS** ŌĆö so most of this phase has no app-owned visual surface.

The app actually owns exactly **one new visual control** this phase:

1. **"Share as Frame" affordance** (D-04) ŌĆö a new control added to the existing Phase-7 receipt-page action row, alongside the existing `SHARE THE RECEIPT ŌåÆ` button.

Everything else is either **inherited/locked** or an **interaction + copywriting contract** rather than a CSS contract:

- The **OG card variants** (Settled / DuelSettled / CallerExited) are reused **unchanged** as the Frame embed `imageUrl` (D-02, criterion 3 visual continuity). They are locked from Phase 7 ŌĆö **not re-specified here.**
- The **Warpcast-rendered Frame buttons** (Follow/Fade/Challenge for live; Follow-person/Challenge/Quote for settled ŌĆö D-02/D-06) are an **interaction + copy contract**: button **labels, ordering, and one-tap min-stake semantics** (D-07). Warpcast controls their pixels; we control their text and order. See [Frame Button Contract](#frame-button-contract-warpcast-rendered--copyorder-only).

The design system (neobrutalist `@call-it/ui`) is **inherited and locked** from prior phases. The tables below record the inherited tokens for checker/executor reference; **this phase introduces no new design tokens.**

## Copywriting Contract

This is the **primary deliverable of this phase's UI work** ŌĆö button labels, ordering, and copy for both the in-app control and the Warpcast-rendered Frame buttons.

### In-app control (receipt page)

| Element | Copy |
|---------|------|
| Primary CTA (new) | `SHARE AS FRAME ŌåÆ` (uppercase, mono, sits to the right of the existing `SHARE THE RECEIPT ŌåÆ` in the action row) |
| CTA behavior | Opens the Warpcast compose intent (`warpcastComposeUrl(receiptUrl, text)` from `@call-it/shared`) in a new tab; `embeds[]=receiptUrl` carries the Mini App embed so Warpcast renders the OG card + Frame buttons. Reuses `buildShareText` (outcome word + handle + statement, Ōēż240 chars). |
| Empty state | not applicable ŌĆö the control only renders on a settled/live receipt that already has data; there is no empty variant. |
| Error state | If `warpcastComposeUrl` cannot be constructed (missing receipt URL / handle), the control is **omitted** (not rendered as a dead button) ŌĆö consistent with the existing `og:image`/share fallback pattern. No inline error string shown to the user. |
| Destructive confirmation | none ŌĆö sharing is non-destructive; no confirmation dialog. |

### Frame Button Contract (Warpcast-rendered ŌĆö copy/order only)

The Frame endpoint selects the button set by **call status** read from the relayer `/api/calls/:id/live-state` (D-02). Warpcast renders the pixels; **we own the labels, the order, and the one-tap transaction semantics.** Button labels are **Title Case** (Farcaster Frame button convention ŌĆö these are NOT in-app neobrutalist mono/uppercase controls).

**LIVE call cast** ŌĆö 3 buttons, in this order:

| # | Label | Action (one-tap tx) | Contract path |
|---|-------|---------------------|---------------|
| 1 | `Follow` | Follow at **minimum stake** ($1 follow), one-tap (D-07) | FollowFadeMarket (Phase 2) |
| 2 | `Fade` | Fade at **minimum fade stake**, one-tap (D-07) | FollowFadeMarket (Phase 2) |
| 3 | `Challenge` | Propose 1v1 challenge | ChallengeEscrow (Phase 3) |

**SETTLED receipt cast** ŌĆö 3 buttons, in this order (D-06; `Fade` dropped ŌĆö market closed):

| # | Label | Action (one-tap tx) | Contract path |
|---|-------|---------------------|---------------|
| 1 | `Follow` | Social-follow the caller's profile | ProfileRegistry (Phase 1.5) |
| 2 | `Challenge` | Propose 1v1 challenge | ChallengeEscrow (Phase 3) |
| 3 | `Quote` | Quote-call the settled receipt | CallRegistry |

**Deep-link affordance (both states):** larger-than-minimum stakes are NOT entered in-frame (avoids unreliable cross-client amount input ŌĆö D-07). The Frame provides an **`Open in Call It`** path that deep-links to the receipt page (`/call/:id`) where the full FollowFadeModal stake input lives. Label: `Open in Call It` (Title Case, Warpcast convention).

**One-tap semantics (D-07):** the in-frame `Follow`/`Fade` transactions construct a **fixed minimum-stake** transaction (no amount input in the Frame). Larger stakes route through `Open in Call It`. This is an interaction contract enforced by the Frame transaction-data builder, not a CSS rule.

**Copy invariants:**
- Outcome word in the cast text (`buildShareText`) must always survive truncation (Ōēż240 chars) ŌĆö locked in `@call-it/shared`; do not re-implement.
- Frame button labels are single words (`Follow` / `Fade` / `Challenge` / `Quote`) plus the multi-word `Open in Call It` ŌĆö keep them short for narrow Warpcast button chrome.
