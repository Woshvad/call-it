Vertical slice A: make a Call It receipt render as a Farcaster Mini App embed. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Farcaster client -> /call/:id HTML head | Reads server-rendered embed meta; embed content must contain only URLs + brand constants, never raw user strings |
| Farcaster client -> /.well-known/farcaster.json | Public unauthenticated read of display-only config |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Spoofing | embed/manifest imageUrl/homeUrl origin | mitigate | Origin derives ONLY from `NEXT_PUBLIC_OG_BASE_URL` (origin-locked); never from request params/headers — an attacker cannot point the embed/manifest at a foreign origin. Acceptance asserts the env grep. |
| T-08-02-02 | Tampering | stale OG embed image on status transition (Pitfall 4) | mitigate | embed `imageUrl` is built from the SAME `statusVersion` already fetched in `generateMetadata`; no second/stale fetch. |
| T-08-02-03 | Cross-site scripting / injection | raw handle/statement rendered into meta content | mitigate | Embed JSON is `JSON.stringify`-escaped and contains ONLY URLs + brand constants; no raw user string is concatenated into the meta tag (RESEARCH Security Domain). |
| T-08-02-04 | Information Disclosure | public manifest leaks internal config | accept | Manifest is public by design and contains only display config (name, URLs, colors); no secrets. `accountAssociation` (a domain proof, not a secret) is deferred to Phase 10. |
| T-08-02-SC | Tampering | npm installs | mitigate | Zero new npm deps — the embed + manifest are plain meta/JSON using only in-repo modules. No `@farcaster/*` ([ASSUMED]) package introduced. |
</threat_model>
