<objective>
GAP 1 (UAT test 1, severity: major — Core Value violation). Close the fake-outcome-word gap: a SETTLED loss (call #14 = CallerLost) currently renders the receipt stamp AND the Farcaster SHARE AS FRAME compose text as "CALLED IT", publicly casting a loss as a win. Only the `/og/[callId]` embed image is correct ("LOUD AND WRONG").

Root cause (from 08-UAT.md): the relayer `GET /api/calls/:id/live-state` does NOT return `outcome` (nor `repDelta`/`fadeRealShare`), so `fetchCallData` always sets `callData.outcome = undefined` → `outcomeWordResult` is `null` (page.tsx:1059) → `outcomeWord` falls back to the hardcoded `'CALLED IT'` (page.tsx:1368), which feeds BOTH the receipt stamp and `buildShareText` (page.tsx:1383). The `og:title` is a third wrong answer ("Live Call") for a settled call.

This is the project Core Value: receipts must be unfakeable. A settled loss must NEVER share as a win, and an unknown outcome must NEVER be fabricated into a positive word.

Purpose: Make the receipt PAGE + SHARE text the single source of truth that already drives the (correct) `/og/[callId]` card — settled outcome (CallerWon/CallerLost), repDelta, and fadeRealShare must reach the page so `getOutcomeWordResult` returns the true §15.7 word.

Output:
- Relayer `/live-state` extended to surface `outcome` (from the on-chain `getCall` struct it already reads) + `repDelta` + `fadeRealShare` (subgraph-sourced, mirroring `/og`'s `getSettledFields`).
- Receipt page: real outcome flows through `getOutcomeWordResult`; the dangerous `?? 'CALLED IT'` default REMOVED and replaced with a fail-safe neutral/loading state.
- `layout.tsx` og:title reflects the settled outcome word.
- Regression test proving CallerLost → "LOUD AND WRONG" (never "CALLED IT").
</objective>
