# Quick Task 260611-h44: Receipt Page Settled-State Honesty Fixes Summary

**Settled receipt stops fabricating wins: zero P&L/rep render muted with no sign, unconfirmed outcomes get a small "Pending result" block instead of the giant stamp, the '#N' fallback is never styled as identity, and SHARE AS FRAME is demoted to a mono text link under the primary.**

## What Was Done

### Task 1 — Four settled-branch honesty fixes (commit `1401d4d`)

1. **Zero-neutral money/rep (locked decision 1):**
   - P&L stat: three-way ternary — `callerPnl === 0n` renders `$0.00` in `var(--text-secondary)` with no sign; positive keeps `+` accent-win; negative keeps `-` accent-loss.
   - Rep Δ stat: color and sign key off the REAL `repDelta` three-way; displayed number stays `displayedRepDelta` (UI-45 count-up animation untouched); `+` prefix only when `repDelta > 0`.
   - Caller rep chip: `accent-win` only when `globalRep > 0`; zero AND negative both render `var(--text-secondary)`.
2. **Neutral PENDING RESULT — no stamp (locked decision 2):** added an `isNeutralOutcome` middle branch between `isDisputed` and the stamp, mirroring the dispute block's structure: `.h-2` heading "Pending result" colored `outcomeStyle.hex` (slate `#94A3B8` for the neutral word) + mono 13px explainer "Settlement recorded — the final outcome hasn't been confirmed yet. Check back shortly." `data-outcome-word={outcomeWord}` preserved on the container div (Playwright hook). Real verdict words keep the full Stamp treatment byte-for-byte unchanged (including `outcomeStyle.sub` + `outcomeLozenge`).
3. **Unresolved-caller byline (locked decision 3, D-07/AUTH-44):** added `const handleResolved = Boolean(callerProfile?.handle ?? callData?.handle)`. When false: neutral `?` avatar (`bg-tertiary`/`text-tertiary`, no grad class seeded by '#14') + mono lowercase "caller unresolved" instead of the 20px/900 display-font handle. `handle` itself unchanged — share-text builders remain gated on `callData?.handle` so they only consume real handles. Sweep confirmed: winners/losers/final-positions tables use per-position `p.handle` (real relayer data); the LIVE branch `displayHandle` fallback untouched per plan scope.
4. **Demoted SHARE AS FRAME (locked decision 4):** action row is now always `flexDirection: 'column'` (`alignItems: isMobile ? 'stretch' : 'flex-end'` keeps the desktop anchor position). The `.btn outline-white` sibling button replaced with a mono text link "or share as a Farcaster frame ↗" (`className="mono receipt-frame-link"`, fontSize 11.5, `text-secondary`, `minHeight: 44` inline-flex, mobile `width: 100%`, `target="_blank" rel="noopener noreferrer"` kept — T-h44-01 reverse-tabnabbing guard). globals.css gained one rule next to `.btn.outline-white`: `.receipt-frame-link:hover { text-decoration: underline; }`.

### Task 2 — Test assertion sweep (commit `abe0a84`)

- Grep sweep confirmed the plan's predictions exactly: 'SHARE AS FRAME' appears only in a comment in `settled-outcome-truth.test.ts` (unit asserts `resolveSettledWord`/`buildShareText` behavior — unchanged, left as-is); no spec asserts the removed outline button.
- `responsive.spec.ts` "outcome word legible" (>=36px on `[data-outcome-word]`): `SEEDED_SETTLED_CALL` defaults to `'14'` (CallerLost → real "LOUD AND WRONG" → stamp path), so the assertion stays valid. Added the planned comment noting the neutral PENDING RESULT state is intentionally exempt — a future `RESPONSIVE_SETTLED_CALL_ID` pointing at a neutral call is a seeding error, not a regression.
- "action row full-width" spec needs no change: the locator (`button, a` inside `[data-receipt-action-row]`) still matches (both controls are anchors), and the demoted link is full-width on mobile by construction.
