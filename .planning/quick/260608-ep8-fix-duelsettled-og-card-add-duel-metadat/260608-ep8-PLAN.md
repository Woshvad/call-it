  <action>
Create `apps/web/app/duel/[challengeId]/layout.tsx` as a Server Component (NO `'use client'`) that mirrors the structure, comment style, and imports of `apps/web/app/call/[id]/layout.tsx`. Use `import type { Metadata } from 'next'` and `import type { ReactNode } from 'react'`. Keep it ~100 lines like the call layout.

Define a `Props` type: `params: Promise<{ challengeId: string }>` (App Router async params) and `children: ReactNode`.

Add a private async helper `fetchDuelStatusVersion(challengeId)` mirroring the call layout's `fetchCallMeta` try/catch-returns-null shape:
  - Read the relayer URL exactly as the call layout does: `process.env['RELAYER_URL'] ?? process.env['NEXT_PUBLIC_RELAYER_URL'] ?? ''`. If empty, return null (skip fetch) so the caller uses the v=0 fallback.
  - `fetch(`${relayerUrl}/api/duels/${challengeId}/live-state`, { next: { revalidate: 60 } })`. On `!res.ok` return null.
  - The relayer DUEL endpoint does NOT return `statusVersion`/`marketLine`/`handle` (per duel-live-state.ts `DuelLiveStateResponse`). It returns a `status` STRING LABEL. Parse `const data = await res.json() as { status?: string }` and map the label to an ordinal via a module-level const map: Proposed=0, Accepted=1, Rejected=2, Refunded=3, Settled=4. Unknown/missing label → 0. Return `{ statusVersion: <ordinal> }`. This ordinal is the D-09 `?v=` cache-buster — it bumps the CDN key as the duel transitions Proposed→Accepted→Settled, matching D-09 intent (mirrors how the call layout bumps on Live→CallerExited→Settled).
  - Wrap the whole body in try/catch; on any throw return null.

Implement `export async function generateMetadata({ params }: Props): Promise<Metadata>`:
  - `const { challengeId } = await params;` (MUST await — App Router async params).
  - `const meta = await fetchDuelStatusVersion(challengeId);`
  - `const statusVersion = meta?.statusVersion ?? 0;`
  - Title: when the fetch succeeded use `Duel #${challengeId} — Call It` (no marketLine is available from the duel endpoint); on fetch failure (meta === null) use the generic `Duel — Call It`.
  - Description (similar to the call layout's tone): `A 1v1 challenge on Call It — winner takes the pot, permanent onchain receipt.`
  - OG image URL: `const ogImageUrl = `/og/duel/${challengeId}?v=${statusVersion}`;` — relative URL is intentional (Vercel injects the deployment origin as metadataBase fallback; confirmed working on /call/[id]). Do NOT inline any contract address, RPC key, or origin host — uses challengeId from route params only.
  - Return a `Metadata` object identical in shape to the call layout: top-level `title` + `description`; `openGraph` with `{ title, description, images: [{ url: ogImageUrl, width: 1200, height: 630, alt: title }], type: 'website' }`; `twitter` with `{ card: 'summary_large_image', title, description, images: [ogImageUrl] }`.

Add the default export: `export default async function DuelLayout({ children }: Props) { return <>{children}</>; }`.
