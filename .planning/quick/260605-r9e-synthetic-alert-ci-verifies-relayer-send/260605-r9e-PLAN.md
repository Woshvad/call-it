<objective>
Fix the synthetic-alert CI self-test to verify the Telegram alert pipeline via the relayer's
SEND-CONFIRMATION response instead of polling Telegram getUpdates.

Root cause (confirmed via the Telegram Bot API this session): TELEGRAM_CHAT_ID_P0 (6070757637)
is the operator's PRIVATE DM with @calllitbot (getChat → type:"private"), not a channel. A bot
cannot read its OWN outgoing messages via getUpdates (only incoming updates + channel posts where
it is an admin), so the daily cron's getUpdates poll could never confirm the nonce → the cron
failed every run, even though the live DM alerting works fine. The relayer handler returns HTTP 200
ONLY after sendAlert() resolves — and sendAlert awaits bot.sendMessage and re-throws on failure
(→ 500). So a relayer 200 + echoed nonce IS proof of delivery; the CI should trust that.
</objective>
