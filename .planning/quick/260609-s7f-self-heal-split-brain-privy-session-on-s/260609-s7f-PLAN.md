<objective>
Make `/signin` self-heal a split-brain / stale Privy session. Privy can report `authenticated===true` from a leftover localStorage session while `getAccessToken()` can no longer mint a valid access token. Today the post-auth effect writes no cookie and does nothing — the user is stranded (bell shows, sign-in buttons no-op because Privy `login()` ignores calls while a session exists, and protected routes bounce back to `/signin` because the middleware never sees a `privy-token` cookie).

This plan keeps the happy path byte-identical (token → set `privy-token` cookie `Max-Age=3600` + `Secure` on https → `router.push('/')`) and ADDS a recovery branch: when authenticated but no token (getAccessToken returns `null` OR throws), clear the cookie (`Max-Age=0`) and call Privy `logout()` — invisible recovery to a clean signed-out `/signin`.

Purpose: Unstick users who hit the stale-session dead-end. The root cause was verified live in-browser; this is a faithful implementation with exact code — do NOT redesign.
Output: A single modified file, `apps/web/app/signin/SignInButtons.tsx`, with exactly two changes: (1) `logout` added to the `usePrivy()` destructure, (2) the post-auth `useEffect` replaced with the self-heal version.
</objective>

Replacement effect (use exactly this, preserving the existing leading comment block above the effect — keep that comment as-is):

  useEffect(() => {
    if (!authenticated) return;
    let cancelled = false;
    void (async () => {
      let token: string | null = null;
      try {
        token = await getAccessToken();
      } catch {
        // getAccessToken threw — treat as a dead session (handled below).
        token = null;
      }
      if (cancelled) return;
      if (token) {
        const secure = window.location.protocol === 'https:' ? '; Secure' : '';
        // Max-Age ~1h matches Privy access-token lifetime; a returning user with
        // a live localStorage session but expired cookie self-heals via the
        // /signin bounce, which re-runs this effect.
        document.cookie = `privy-token=${token}; path=/; SameSite=Lax; Max-Age=3600${secure}`;
        router.push('/');
      } else {
        // Split-brain recovery: Privy reports authenticated===true from a leftover
        // localStorage session, but getAccessToken can no longer mint a valid access
        // token. The old code did nothing here, stranding the user (bell shows,
        // login() no-ops while a session exists, protected routes bounce to /signin).
        // Clear the stale cookie and log out so the user lands on a clean signed-out
        // /signin. After logout, authenticated→false and this effect early-returns —
        // no loop, and logout does NOT trigger login().
        const secure = window.location.protocol === 'https:' ? '; Secure' : '';
        document.cookie = `privy-token=; path=/; SameSite=Lax; Max-Age=0${secure}`;
        try {
          await logout();
        } catch {
          // logout failed — leave the user on /signin rather than loop.
        }
      }
    })();
    return () => {
      cancelled = true;
    };
  }, [authenticated, getAccessToken, logout, router]);
