<objective>
Wire up the Tailwind CSS pipeline in `apps/web` so the neobrutalist design system actually renders. Today the pipeline is dead in two independent ways:

1. `apps/web/app/globals.css` has NO `@tailwind` directives, so Tailwind utilities are never emitted.
2. `globals.css` is imported only from `apps/web/app/page.tsx` (the `/` route), not the root layout, so even the hand-written keyframes only load on `/`.

Without a `postcss.config.js` registering the `tailwindcss` plugin, the `@tailwind` directives are passed through uncompiled and emit zero utilities — the verify gate below would return 0 matches and fail. `tailwind.config.ts` is already correct and is NOT touched.

Purpose: turn on the entire shared `@call-it/ui` design system (every `<Button>`/`<Card>`/`<Stamp>`/etc.) app-wide for the first time. The deployed `/signin` HTML currently has zero `<link rel="stylesheet">`.
Output: 5 files modified/created inside `apps/web` only; a `next build` that emits a CSS bundle containing real Tailwind utilities.
</objective>

## REQUIRED scope addition beyond the original 3-file diagnosis — read before executing

The task brief scoped this to 3 files and said "tailwind.config.ts is already correct — DO NOT change" (true). But planning verified that the stated 3-file change CANNOT make the verify gate pass, because the Tailwind PostCSS plugin is not wired anywhere:

- There is NO `postcss.config.{js,cjs,mjs,ts}` in `apps/web`, in the repo root, or in `packages/config`. Confirmed by filesystem scan.
- `autoprefixer` and `postcss` are NOT declared in `apps/web/package.json` (they exist in the pnpm store only as transitive deps).
- Next.js 16.2.6's built-in PostCSS loader (`next/dist/build/webpack/config/blocks/css/plugins.js` → `getPostCssPlugins` → `getDefaultPlugins`) falls back, when no `postcss.config` is found, to ONLY `postcss-flexbugs-fixes` + `postcss-preset-env`. It does NOT inject `tailwindcss`. Verified by reading the installed Next source.
- Current emitted bundles confirm zero Tailwind output today: `apps/web/.next/static/css/65d232eff0c4a92d.css` (284b) is exactly the hand-written `cardEnter` keyframes; `7455b2ac48aa0b6c.css` (3571b) is a vanilla-extract atomic system (`._1n3pr300{...}`), NOT Tailwind (no `--tw-` tokens).

Therefore Task 1 below creates `apps/web/postcss.config.js` (tailwindcss + autoprefixer) and adds the two devDependencies. This stays strictly inside `apps/web`. It does NOT modify `tailwind.config.ts`, `packages/ui`, or any postcss config elsewhere (there are none to modify). It is the minimum required to satisfy the user-specified verify gate.

This is reachability completion, not scope creep: the original goal ("Tailwind utilities emitted; grep for `bg-brand-accent`/`e8f542`/`--tw-`/`4px_4px`/`.uppercase` finds them") is impossible without it.
</scope_note>
