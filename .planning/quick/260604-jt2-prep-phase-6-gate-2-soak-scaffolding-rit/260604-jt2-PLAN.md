  Search for the literal string "42161" in lines that also contain "chainId" OR "domain" (case-insensitive match on the line).
  PASS if ≥1 such line found. FAIL if 0 matches.
  Implementation: after reading each file, split on newlines, filter lines containing "42161",

Gate-c — "Relayer ETH balance ≥ 0.5 ETH on Arbitrum Sepolia":
  Read RELAYER_ADDRESS and ARBITRUM_SEPOLIA_RPC_URL from process.env.
  If either is absent: print "[gate-c] SKIPPED — RELAYER_ADDRESS or ARBITRUM_SEPOLIA_RPC_URL not set" and count as SKIPPED (not FAIL).
  If both present: createPublicClient({ chain: arbitrumSepolia, transport: http(rpcUrl) }).getBalance({ address: relayerAddress as \`0x\${string}\` }).
  PASS if balance >= parseEther('0.5'). FAIL otherwise with actual balance in ETH.
  Import { parseEther, formatEther } from 'viem' for this.

Gate-d — "Pyth bytes32 feed IDs match Hermes API for BTC/ETH/SOL/ARB/OP/POL":
  For each of the 6 symbols in order [BTC, ETH, SOL, ARB, OP, POL]:
    Fetch https://hermes.pyth.network/v2/price_feeds?query=Crypto.{SYM}/USD using global fetch (Node 18+ built-in).
    Parse the JSON array. Find the entry where id matches PYTH_FEED_IDS[SYM] (without the 0x prefix — Hermes returns IDs without 0x).
    PASS for that symbol if found. FAIL if no entry's id equals the constant (strip 0x from the constant for comparison).
    On network error or timeout: mark that symbol as SKIPPED with a note (do not hard-fail the whole gate on network issues — gate-d failures are advisory since Hermes may be unavailable in CI).
  Gate-d overall: PASS if all 6 symbols pass, FAIL if any symbol returns a mismatch (not SKIPPED), SKIPPED if all are skipped.
