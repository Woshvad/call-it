# quick-260611-h36: Relayer Resilience Overhaul Summary

LIVE OUTAGE response (2026-06-11). Upstash quota exhausted (500K/500K, resets ~Jul 1) → zero caching → The Graph Studio 429-storms → feed/profile/positions empty despite 15 on-chain calls. Auto-settlement was structurally inert (no enqueueSettlement producer, throw-stub walletClient, dead Redis for BullMQ), and the idle BullMQ worker polled Upstash 24/7 (prime quota-burn suspect). Call #15 (ETH>=$2,300, expires 2026-06-12 10:08 UTC) had nothing to settle it.

## What was built

1. **L1 memory cache (commit e03c6bd):** `MemoryCache` bounded TTL map (FIFO eviction, lazy expiry — zero idle CPU, no timers) + `getCached/setCached` L1-first adapter. All 8 Redis cache sites converted (keys/TTLs/response shapes byte-identical). Single-Fly-machine rationale documented in the module header.
2. **5 hard-500 Redis paths degraded (same commit):** dup-check skips cache → on-chain check; paymaster POST fails CLOSED (deny, exact bundler JSON-RPC shape — T-h36-02); paymaster-count GET → `200 {count:null, capacity:5, remaining:null, degraded:true}`; unlink-purge Redis del best-effort (still 200, watcher backstop); synthetic nonce falls back to a dedicated in-memory `MemoryCache(1000)` (replay guard preserved, single-machine sound — T-h36-03); admin cap PATCH → `503 {error:'cache_unavailable'}`.
3. **Subgraph circuit breaker (commit cf4d844):** `executeQuery` → breaker-wrapped (opens on ANY failure incl. 429/timeout; 120s cooldown, env `SUBGRAPH_BREAKER_COOLDOWN_MS`; implicit half-open probe). Last-known-good snapshots (dedicated `MemoryCache(200)`, sha1(query+variables) keys, 1h hard horizon). Feed responses carry `_source:'stale'` + `x-source: stale` when snapshot-served (wire shape unchanged). New breaker-routed `queryAcceptedChallengeIds` export.
4. **Settlement poller (commit 8937aa0):** Redis-free `setInterval` worker — ascending `getCall` frontier discovery (stops at zeroed tuple, max 50 probes/tick), expired Live/CallerExited Pyth candidates settled via `settlePythCall` (challenge ids through the breaker, [] fallback). 30-attempt cap → `settle_failed` P0 once + 10-min backoff. Non-Pyth calls: one structured skip log per call. BullMQ watcher gated behind `SETTLEMENT_BULLMQ_ENABLED` (default false).

## Deviations from Plan

**1. [Minor] `getCached` takes an optional `backfillTtlSeconds` parameter**
- The plan's getCached signature had no TTL, but its body spec required backfilling the L1 with `ttlSeconds*1000` on a Redis hit. Resolved by an optional second parameter — call sites pass their site TTL. Artifact contract (exports `getCached`/`setCached`) unchanged.

**2. [Rule 3 - Test compat] `__tests__/ens-resolver.test.ts` reworked to module-level redis mock**

No assertion semantics weakened — same 8 tests, same behaviors, format updated per the plan's explicit format change. `resolveEns(address, redis)` signature kept (param `_redis` deprecated) so call sites/tests compile unchanged.

**3. [Part C, anticipated] `memoryCache._clearAllForTesting()` added to 6 test files' beforeEach**

## OPERATOR NOTES (first deploy expectations)

**New env flags (both optional — defaults are the safe outage-response posture):**

| Flag | Default | Meaning |
|------|---------|---------|
| `SETTLEMENT_POLLER_ENABLED` | `true` (ON unless set to `false`) | Redis-free chain-polling settlement poller |
| `SETTLEMENT_BULLMQ_ENABLED` | `false` (OFF unless set to `true`) | Old BullMQ settlement watcher (dead producer + 24/7 Upstash polling — leave OFF) |
| `SETTLEMENT_POLLER_INTERVAL_MS` | `60000` | Poller tick interval |
| `SUBGRAPH_BREAKER_COOLDOWN_MS` | `120000` | Breaker open window |

**Optional secret — `SETTLEMENT_SIGNER_PRIVATE_KEY`:**
- Absent (current state) → the poller runs **IDLE dry-run**: one P1 Telegram alert at boot ("poller IDLE: no settlement signer configured") + a `settlement_poller_would_settle` warn log per expired call per tick. **Zero transactions are sent.**
- Set it as a Fly secret (`flyctl secrets set SETTLEMENT_SIGNER_PRIVATE_KEY=0x... -a call-it-relayer-sepolia`) to enable live settling. The key is pino-redacted, never logged, never committed; a malformed value logs `settlement_signer_invalid` (sanitized) and falls back to IDLE.
- The signer wallet needs Sepolia ETH for gas; the SM contract pays the Pyth update fee from its own balance (low-balance fires the existing `settle_failed` ETH alert).

**What to expect in logs on first deploy:**
- `settlement_watcher_bullmq_disabled` (BullMQ no longer polls Upstash — quota burn stops)
- `settlement_poller_started` (`idle: true` until the secret is set) + the single P1 IDLE alert
- `settlement_poller_would_settle` for call #15 once it expires (2026-06-12 10:08 UTC) — proof the discovery rail works before arming the signer
- `subgraph_breaker_opened` / `subgraph_breaker_stale_served` while Studio 429s — feed serves `_source:'stale'` instead of empty
- `cache_redis_read_failed` / `cache_redis_write_failed` warns while the Upstash quota is exhausted — responses still serve from the L1
- `settlement_poller_skip_non_pyth` once per non-Pyth call (known criteria-store gap, no alert)
