<objective>
Fix the deployed Sepolia relayer's GET /api/profile/:address indefinite hang (>70s, curl exit 28 on every address) by bounding every upstream dependency so the route ALWAYS responds within a hard deadline, plus regression tests.

Diagnosis is already complete (orchestrator, 2026-06-10, from Fly logs + source — trust it, do not re-investigate): the route's Promise.allSettled has two viem readContract legs against Arbitrum Sepolia whose transport falls through unset env vars to the public `https://sepolia-rollup.arbitrum.io/rpc`, which tarpit-throttles — viem's default transport timeout/retries never settle the legs, so the request never completes. Compounding context: Upstash free-tier quota is exhausted, so every Redis command may reject, and `resolveEns` calls `redis.get`/`redis.set` UNGUARDED.

Purpose: /api/profile is on the critical path for every profile render; an unbounded hang is a self-inflicted DoS.
